# HDC-008：Fast Memory地址验证不足漏洞

## 漏洞概述

| 属性 | 值 |
|------|-----|
| **漏洞 ID** | HDC-008 |
| **类型** | DMA安全 (DMA Security) |
| **CWE** | CWE-119: Improper Restriction of Operations within Bounds |
| **严重性** | High |
| **置信度** | 85% |
| **模块** | hdc_communication |
| **文件** | src/sdk_driver/hdc/pcie/common/hdcdrv_core.c |
| **行号** | 2904-2953 |
| **函数** | hdcdrv_get_fast_mem_info |

### 描述

来自SQ描述符的Fast Memory地址用于查找内存节点，缺乏完整验证。src_data_addr、dst_data_addr、src_ctrl_addr、dst_ctrl_addr来自设备固件，可能指向无效内存。恶意设备固件可能导致DMA到任意地址。

// 在访问前验证session索引和地址
STATIC int hdcdrv_get_fast_mem_info_safe(...)
{
    // 新增: 验证session索引
    if (sq_desc->remote_session >= HDCDRV_MAX_SESSION) {
        return HDCDRV_PARA_ERR;
    }
    
    session = &hdc_ctrl->sessions[sq_desc->remote_session];
    
    // 新增: 验证session状态
    if (hdcdrv_get_session_status(session) != HDCDRV_SESSION_STATUS_CONN) {
        return HDCDRV_SESSION_NOT_CONN;
    }
    
    // 新增: 验证地址范围
    if (!validate_fast_mem_address(sq_desc->src_data_addr, sq_desc->data_len)) {
        return HDCDRV_PARA_ERR;
    }
    
    // 验证通过后继续处理
}
