# SVM-001：缺失授权检查漏洞

## 漏洞概述

| 属性 | 值 |
|------|-----|
| **漏洞 ID** | SVM-001 |
| **类型** | 缺失授权检查 (Missing Authorization Check) |
| **CWE** | CWE-285: Improper Authorization |
| **严重性** | High |
| **置信度** | 92% |
| **模块** | svm_memory |
| **文件** | src/sdk_driver/svm/v2/common/svm_module_ops.c |
| **行号** | 439-475 |
| **函数** | devmm_svm_ioctl |

### 描述

ioctl处理函数 devmm_svm_ioctl 接收用户命令时缺乏能力检查（capability check）。该模块仅依赖进程ID（tgid/pid）验证，这对于内核模块安全是不充分的。攻击者可能在没有适当授权的情况下调用内存操作。

## 修复建议

### 立即修复方案

```c
// 1. 添加能力检查
STATIC long devmm_svm_ioctl(ka_file_t *file, u32 cmd, unsigned long arg)
{
    // 新增: 权限检查
    if (!capable(CAP_SYS_ADMIN) && !capable(CAP_SYS_RAWIO)) {
        devmm_drv_err("SVM ioctl requires elevated privileges (cmd=0x%x)\n", cmd);
        return -EPERM;
    }

    // 新增: 特定命令的能力检查
    switch (cmd_id) {
        case DEVMM_SVM_CMD_MEM_ALLOC:
        case DEVMM_SVM_CMD_MEM_SHARE:
            if (!capable(CAP_SYS_ADMIN)) {
                return -EPERM;
            }
            break;
        case DEVMM_SVM_CMD_MEM_IMPORT:
            // 验证进程有权限导入目标内存
            if (!verify_import_permission(file, cmd)) {
                return -EACCES;
            }
            break;
    }

    // 现有验证...
    if ((_KA_IOC_TYPE(cmd) != DEVMM_SVM_MAGIC) || ...) {
        return -EINVAL;
    }

    ret = devmm_dispatch_ioctl(file, cmd, &buffer);
}
```

### 进程绑定验证

```c
// 2. 验证进程与设备的绑定关系
STATIC int verify_svm_process_binding(ka_file_t *file, u32 cmd)
{
    struct devmm_svm_process *svm_proc = ka_fs_get_file_private_data(file);
    
    // 验证进程ID匹配
    if (svm_proc->process_id.hostpid != current->tgid) {
        return -EACCES;
    }
    
    // 验证namespace匹配
    if (!namespace_match(current->nsproxy, svm_proc->nsproxy)) {
        return -EACCES;
    }
    
    return 0;
}
```

### 长期修复方案

1. **分级权限**: 不同命令要求不同能力级别
2. **进程绑定**: 强制进程与设备绑定后才能执行操作
3. **审计日志**: 记录所有ioctl操作用于审计
4. **资源配额**: 为每个进程设置内存操作配额
