# HDC-001: 设备控制消息验证不足漏洞

## 漏洞概述

| 属性 | 值 |
|------|-----|
| **漏洞 ID** | HDC-001 |
| **类型** | 输入验证不足 (Input Validation) |
| **CWE** | CWE-20: Improper Input Validation |
| **严重性** | Critical |
| **置信度** | 90% |
| **模块** | hdc_communication |
| **文件** | src/sdk_driver/hdc/pcie/common/hdcdrv_core.c |
| **行号** | 4151-4214 |
| **函数** | hdcdrv_ctrl_msg_recv |

### 描述

设备侧固件控制消息处理缺乏全面验证。来自设备固件（semi_trusted）的消息仅经过基本长度检查后即被解析和处理，缺乏完整性和内容完整性验证。恶意设备固件可发送构造的控制消息导致内存损坏或权限提升。

### 验证缺失分析

**hdcdrv_ctrl_edge_check 仅验证:**
- `devid` 范围检查 (line 4050)
- `in_data_len` 最小长度检查 (line 4051)
- `phy_addr_num` 上限检查 (line 4062-4065)

**缺失验证:**
- 消息内容完整性 (无CRC校验之外的验证)
- 消息字段语义正确性
- unique_val 的加密绑定
- 连接请求的真实性

## 利用条件

### 触发条件

| 条件 | 描述 |
|------|------|
| **攻击者位置** | NPU 设备固件 (通过 PCIe DMA 发送控制消息) |
| **信任边界** | 跨越 PCIe Hardware Interface 边界 |
| **前置条件** | 设备固件已被攻陷或存在恶意固件 |
| **触发方式** | 发送构造的控制消息到 hdcdrv_ctrl_msg_recv |

### 攻击者能力要求

- **能力等级**: Semi-trusted (设备固件)
- **所需权限**: 设备固件控制权
- **技术要求**: 
  - 了解 HDC 控制消息协议格式
  - 能够构造有效的消息结构
  - 了解 session 管理机制

### 利用场景

1. **恶意固件植入**: 攻击者通过供应链攻击植入恶意固件
2. **固件漏洞利用**: 通过固件自身的漏洞获得执行控制
3. **物理攻击**: 通过硬件调试接口修改固件行为

## 影响评估

### 直接影响

| 影响类型 | 严重性 | 描述 |
|----------|--------|------|
| **内存损坏** | Critical | 恶意消息可导致内核内存损坏 |
| **权限提升** | Critical | 通过控制 session 分配实现权限绑定绕过 |
| **会话劫持** | High | 通过伪造 unique_val 劫持已建立会话 |
| **拒绝服务** | High | 发送无效消息导致服务崩溃 |

### 潜在后果

1. **内核崩溃**: 恶意消息导致内核态代码执行异常
2. **数据泄露**: 通过会话劫持读取其他进程的数据
3. **权限绑定绕过**: 绕过进程-设备绑定检查
4. **系统不稳定**: 频繁发送构造消息导致资源耗尽
