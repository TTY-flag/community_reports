# HDC-003: SQ描述符验证不足漏洞

## 漏洞概述

| 属性 | 值 |
|------|-----|
| **漏洞 ID** | HDC-003 |
| **类型** | 输入验证不足 (Input Validation) |
| **CWE** | CWE-125: Out-of-bounds Read |
| **严重性** | High |
| **置信度** | 85% |
| **模块** | hdc_communication |
| **文件** | src/sdk_driver/hdc/pcie/common/hdcdrv_core.c |
| **行号** | 2076-2166 |
| **函数** | hdcdrv_msg_chan_recv_handle |

### 描述

来自设备的SQ描述符处理缺乏充分验证。local_session和remote_session字段来自设备固件，直接用于访问session结构而缺乏完整的边界和有效性检查。恶意设备可能引用无效session导致数组越界访问。

## 修复建议

### 立即修复方案

```c
// 在数组访问前验证session索引
STATIC int hdcdrv_msg_chan_recv_handle(...)
{
    struct hdcdrv_session *session = NULL;
    
    // 新增: 先验证session索引范围
    if (rx_desc->local_session >= HDCDRV_MAX_SESSION) {
        hdcdrv_err("Invalid session index from device (local_session=%d)\n",
                   rx_desc->local_session);
        return HDCDRV_PARA_ERR;
    }
    
    // 新增: 验证session状态
    session = &hdc_ctrl->sessions[rx_desc->local_session];
    if (hdcdrv_get_session_status(session) != HDCDRV_SESSION_STATUS_CONN) {
        hdcdrv_err("Session not connected (session=%d)\n", rx_desc->local_session);
        return HDCDRV_SESSION_NOT_CONN;
    }
    
    // 验证通过后再使用
    service = session->service;
    // ...
}
```

### 长期修复方案

1. **所有设备输入先验证**: 建立统一的设备输入验证框架
2. **边界检查宏**: 使用宏自动检查数组索引
3. **设备信任级别**: 根据设备状态调整信任级别
