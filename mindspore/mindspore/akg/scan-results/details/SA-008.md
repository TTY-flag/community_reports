# SA-008: Code Injection via dlopen() in AKGAscendLaunch.cpp

## 漏洞概述

**漏洞类型**: 代码注入 (Code Injection / Arbitrary Code Execution)  
**CWE**: CWE-94 (Improper Control of Generation of Code), CWE-426 (Untrusted Search Path)  
**严重性**: Critical  
**置信度**: 75/100

AKG Ascend Launch Runtime 使用 `dlopen()` 加载动态库（.so 文件），库路径由 Python binding 参数 `path` 和 `kernel_name` 直接拼接构造。若攻击者能控制这些参数，可加载恶意 .so 文件实现任意代码执行。

---

## 源码分析

### 漏洞代码: AKGAscendLaunch.cpp (第149-154行)

```cpp
void akg_ascend_run(std::string path, std::string kernel_name, int device_id, bool is_dynamic,
                    bool use_mem_pool, const py::args &args) {
    // ...
    if (is_dynamic) {
        use_mem_pool = true;
        std::string so_path = path + "/lib" + kernel_name + ".so";  // ← 路径拼接，无验证
        void *handle = dlopen(so_path.data(), RTLD_LAZY);            // ← 加载任意 .so
        if (!handle) {
            std::cerr << "Failed to load library: " << dlerror() << std::endl;
            return;
        }
        // 从加载的库中获取函数指针并执行
        get_tiling_struct_size_function_t get_tiling_struct_size_function =
            (get_tiling_struct_size_function_t)dlsym(handle, tiling_struct_size_name.data());
        tiling_struct_size = get_tiling_struct_size_function();  // ← 执行库中代码
```

### 直接攻击（需 Python 代码控制）

若攻击者能执行 Python 代码（如通过 SA-001），可直接调用 `akg_ascend_run` 加载恶意 .so：

```

                              ↓
┌─────────────────────────────────────────────────────────────────────┐
│  2. 恶意脚本创建恶意 .so 文件                                         │
│     - 编译包含恶意代码的 C/C++ 共享库                                 │
│     - 放置在可控路径（如 /tmp/malicious/）                           │
└─────────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────────┐
│  3. 恶意脚本调用 akg_ascend_run                                      │
│     akgAscendLaunch.akg_ascend_run(                                 │
│         path="/tmp/malicious",                                      │
│         kernel_name="exploit",                                      │
│         is_dynamic=True,                                            │
│         ...                                                         │
│     )                                                               │
└─────────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────────┐
│  4. C++ 代码执行 dlopen                                              │
│     so_path = "/tmp/malicious/libexploit.so"                        │
│     dlopen("/tmp/malicious/libexploit.so", RTLD_LAZY)               │
│     → 加载恶意 .so                                                   │
│     → dlsym() 获取函数指针                                           │
│     → 执行恶意 C/C++ 代码                                            │
└─────────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────────┐
│  5. 任意代码执行（C/C++ 层级）                                        │
│     - 恶意 .so 的 init 函数自动执行                                  │
│     - 可执行任意 native 代码                                         │
│     - 绕过 Python沙箱限制                                            │
└─────────────────────────────────────────────────────────────────────┘
```

## 修复建议

### 1. 路径白名单

```cpp
#include <filesystem>

bool is_valid_kernel_path(const std::string& path) {
    // 限制路径必须在预期目录内
    std::filesystem::path p(path);
    std::filesystem::path allowed_base("/var/lib/akg/kernels");  // 预期目录
    
    // 检查路径是否在允许范围内
    auto abs_path = std::filesystem::canonical(p);
    auto abs_base = std::filesystem::canonical(allowed_base);
    
    if (abs_path.string().find(abs_base.string()) != 0) {
        return false;
    }
    
    // 检查路径穿越
    if (path.find("..") != std::string::npos) {
        return false;
    }
    
    return true;
}

void akg_ascend_run(std::string path, std::string kernel_name, ...) {
    if (!is_valid_kernel_path(path)) {
        std::cerr << "Invalid kernel path: " << path << std::endl;
        return;
    }
    // 原有逻辑...
}
```

### 2. kernel_name 验证

```cpp
bool is_valid_kernel_name(const std::string& name) {
    // 只允许字母、数字、下划线
    for (char c : name) {
        if (!isalnum(c) && c != '_') {
            return false;
        }
    }
    return true;
}
```
