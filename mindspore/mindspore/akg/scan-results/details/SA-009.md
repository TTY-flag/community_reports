# SA-009: Code Injection via dlopen() in GetKernelFunc

## 漏洞概述

**漏洞类型**: 代码注入 (Code Injection / Arbitrary Code Execution)  
**CWE**: CWE-94 (Improper Control of Generation of Code), CWE-426 (Untrusted Search Path)  
**严重性**: Critical  
**置信度**: 75/100

与 SA-008 相同的漏洞模式，但触发点为 `GetKernelFunc()` 函数。`AscendKernelRuntime::RunOpImpl()` 调用 `GetKernelFunc()` 加载动态库，路径同样由参数直接拼接。

---

## 源码分析

### 漏洞代码: AKGAscendLaunchRuntime.cpp (第168-179行)

```cpp
void *AscendKernelRuntime::GetKernelFunc(const std::string &path, const std::string &kernel_name,
                                          const std::string &func_name) {
    std::string file_str = path + "/lib" + kernel_name + kBinFileSuffix;  // ← ".so"
    void *handle = dlopen(file_str.c_str(), RTLD_LAZY | RTLD_LOCAL);      // ← 加载任意 .so
    CHECK(handle != nullptr) << "dlopen failed, file: " << file_str << ", Error:" << dlerror();
    
    std::string func_str = func_name + kDoBinFileSuffix;
    void *func = dlsym(handle, func_str.c_str());  // ← 获取函数指针
    CHECK(func != nullptr) << "dlsym failed, symbol: " << func_str;
    cce_handle_ = handle;
    return func;
}
```

### 调用链: RunOpImpl (第328-350行)

```cpp
void AscendKernelRuntime::RunOpImpl(const std::string &path, const std::string &kernel_name, ...) {
    // 初始化设备
    if (!Init()) { LOG(FATAL) << "Kernel runtime init error."; }
    
    // ...
    
    // 执行内核
    if (!Run(path, kernel_name, is_dynamic, input_tensors, ...)) {
        LOG(FATAL) << "Kernel runtime run error.";
    }
}

bool AscendKernelRuntime::Run(const std::string &path, const std::string &kernel_name, ...) {
    // ...
    auto func_ptr = reinterpret_cast<CallFunc>(GetKernelFunc(path, kernel_name, func_name));
    func_ptr(blockdim, nullptr, stream(), runtimeargs.data());  // ← 执行加载的函数
}
```

### 数据流

```
akg_ascend_run(path, kernel_name) → AscendKernelRuntime.RunOpImpl(path, kernel_name) → Run() → GetKernelFunc(path, kernel_name) → dlopen(path + "/lib" + kernel_name + ".so")
```

## 攻击路径

与 SA-008 相同，攻击者需要:
1. 通过 SA-001 获得Python 执行能力
2. 创建恶意 .so 文件
3. 调用 `akg_ascend_run` 或篡改正常编译输出

```
┌─────────────────────────────────────────────────────────────────────┐
│  1. SA-001 → Python RCE                                              │
└─────────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────────┐
│  2. 创建恶意 .so → 放置在可控路径                                      │
└─────────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────────┐
│  3. 调用 akg_ascend_run(path="/malicious", kernel_name="exploit")   │
└─────────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────────┐
│  4. RunOpImpl → Run → GetKernelFunc                                  │
│     file_str = "/malicious/libexploit.so"                           │
│     dlopen("/malicious/libexploit.so")                              │
└─────────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────────┐
│  5. 恶意 .so 的 constructor 执行                                      │
│     → 任意 native 代码执行                                            │
└─────────────────────────────────────────────────────────────────────┘
```
