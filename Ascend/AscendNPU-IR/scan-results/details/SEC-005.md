## 1. 漏洞根因分析

### 1.1 委托执行机制

`bishengir-opt` 工具实现了目标平台委托机制，当检测到特定目标架构时，将执行委托给另一个二进制文件：

**漏洞代码位置**: `bishengir/tools/bishengir-opt/bishengir-opt.cpp` (第 76-89 行)

```cpp
static int runBishengirOptA5(int argc, char **argv) {
  llvm::SmallVector<llvm::StringRef> arguments;
  arguments.push_back(""); // placeholder, replaced by execute with full path
  for (int i = 1; i < argc; ++i) {
    llvm::StringRef arg(argv[i]);
    if (isAscend910_95TargetArg(arg))
      continue; // skip --target and -target options when delegating to A5
    arguments.push_back(arg);
  }
  if (failed(bishengir::execute("bishengir-opt-a5",
                                bishengir::getBiShengInstallPath(), arguments)))
    return EXIT_FAILURE;
  return EXIT_SUCCESS;
}
```

**触发条件** (第 96-101 行):

```cpp
int main(int argc, char **argv) {
  // If --target=Ascend910_95* or --target=Ascend950* is specified, delegate to
  // bishengir-opt-91095.
  // TODO: this will be removed after bishengir-opt and bishengir-opt-a5

    return runBishengirOptA5(argc, argv);

```

**目标架构检测** (第 54-65 行):

```cpp
static bool isAscend910_95TargetArg(llvm::StringRef arg) {
  llvm::StringRef target;

  else if (arg.starts_with("-target="))
```

### 1.2 execute() 函数的不安全路径搜索

**漏洞代码位置**: `bishengir/lib/Tools/Utils/Utils.cpp` (第 175-221 行)

```cpp
LogicalResult bishengir::execute(StringRef binName, StringRef installPath,
                                 SmallVectorImpl<StringRef> &arguments,
                                 std::optional<llvm::StringRef> outputFile,
                                 unsigned timeoutSeconds) {
  std::string binPath;
  // 第一阶段：从 installPath 搜索（可被环境变量控制）

    }

  // 直接执行找到的二进制，无任何验证

  }
}
```

### 1.3 getBiShengInstallPath() 环境变量信任

**漏洞代码位置**: `bishengir/lib/Tools/Utils/Utils.cpp` (第 285-300 行)

```cpp
std::string bishengir::getBiShengInstallPath() {
  const char *kBiShengInstallPathEnv = "BISHENG_INSTALL_PATH";
  const char *kBiShengInstallPath = getenv(kBiShengInstallPathEnv);
  if (!kBiShengInstallPath) {
    LLVM_DEBUG(llvm::dbgs() << "[DEBUG] BISHENG_INSTALL_PATH is not set.\n");
    return "";
  }
  
  // 无任何验证，直接使用环境变量值
  llvm::SmallString<128> path;
  path.append(kBiShengInstallPath);
  std::error_code errorCode = llvm::sys::fs::make_absolute(path);
  if (errorCode)
    llvm::report_fatal_error("Failed to get absolute path for " + path +
                             " please verify its validity");
  return path.str().str();
}
```

### 1.4 根因总结

| 根因层次 | 问题描述 |
|----------|----------|
| **设计缺陷** | 委托执行模式依赖外部二进制，缺乏信任边界验证 |
| **输入验证缺失** | `BISHENG_INSTALL_PATH` 环境变量未经验证直接使用 |
| **路径搜索不安全** | `llvm::sys::findProgramByName()` 搜索顺序可被攻击者控制 |
| **回退机制危险** | 当 `BISHENG_INSTALL_PATH` 未设置时回退到 PATH 搜索 |
| **完整性验证缺失** | 找到的二进制文件无签名/哈希验证 |

### 5.1 立即修复方案 (Critical)

#### 方案 A: 基于可执行文件路径的相对查找

```cpp
LogicalResult bishengir::execute(StringRef binName, StringRef installPath,
                                 SmallVectorImpl<StringRef> &arguments,
                                 std::optional<llvm::StringRef> outputFile,
                                 unsigned timeoutSeconds) {
  std::string binPath;
  
  // 优先使用可执行文件所在目录
  std::string exeDir = getExecutableDir(); // 新增函数
  
  // 1. 首先在可执行文件同目录查找
  if (auto binPathOrErr = llvm::sys::findProgramByName(binName, {exeDir})) {
    binPath = binPathOrErr.get();

  // 2. 仅当 BISHENG_INSTALL_PATH 在白名单内时使用
  if (binPath.empty() && !installPath.empty()) {
    if (isPathAllowed(installPath)) { // 白名单验证
```

#### 方案 B: 硬编码路径验证

```cpp
std::string bishengir::getBiShengInstallPath() {
  const char *kBiShengInstallPathEnv = "BISHENG_INSTALL_PATH";
  const char *kBiShengInstallPath = getenv(kBiShengInstallPathEnv);
  
  if (!kBiShengInstallPath) {
    // 返回编译时确定的默认路径
    return BISHENG_DEFAULT_INSTALL_PATH; // 编译时常量
  }
  
  // 验证路径合法性
  llvm::SmallString<128> path;
  path.append(kBiShengInstallPath);
  
  // 必须是绝对路径
  std::error_code errorCode = llvm::sys::fs::make_absolute(path);
  if (errorCode) {
    llvm::errs() << "[ERROR] Invalid BISHENG_INSTALL_PATH: " << path << "\n";
    return "";
  }
  
  // 路径规范化
  llvm::sys::path::remove_dots(path, /*removedotdot=*/true);
  
  // 安全检查：禁止路径遍历
  if (path.contains("..") || path.contains("~")) {
    llvm::errs() << "[ERROR] BISHENG_INSTALL_PATH contains unsafe characters\n";
    return "";
  }
  
  // 白名单检查（可选）
  static const std::vector<std::string> allowedPrefixes = {
    "/usr/local/bisheng",
```

## 总结

SEC-005 是一个高危的 CWE-426 (Untrusted Search Path) 漏洞，存在于 `bishengir-opt` 工具的目标平台委托机制中。攻击者可以通过控制 `BISHENG_INSTALL_PATH` 环境变量或 PATH 环境变量，在 IR 优化过程中执行恶意代码，导致代码执行、模型窃取和数据泄露等严重后果。

**关键修复点**:
1. 移除对 `BISHENG_INSTALL_PATH` 的无条件信任
2. 禁止回退到 PATH 搜索
3. 实现二进制路径白名单
4. 添加二进制完整性验证
5. 记录审计日志
