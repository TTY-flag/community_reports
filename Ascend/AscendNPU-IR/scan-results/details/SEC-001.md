# SEC-001: BISHENG_INSTALL_PATH 环境变量劫持漏洞

## 漏洞概览

| 属性 | 值 |
|------|-----|
| **漏洞ID** | SEC-001 (及相关: SEC-002/003/004/005, DF-002/003/004/005/006, VULN-DF-ENV-001) |
| **CWE分类** | CWE-426: Untrusted Search Path |
| **严重性** | **Critical** → Verified: **High** |
| **置信度** | 80 (CONFIRMED) |
| **影响模块** | ToolsLib, Tools, PythonWheel |
| **CVSS 3.1估算** | 7.8 (High) - AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:N |

## 漏洞描述

BiShengIR 编译器通过 `getBiShengInstallPath()` 函数读取 `BISHENG_INSTALL_PATH` 环境变量，并将其用于定位外部二进制可执行文件（`hivmc`, `bishengir-compile-a5`, `bishengir-opt-a5`）。由于该环境变量**没有任何验证、白名单检查或签名校验**，攻击者可以通过设置恶意路径并放置伪造的二进制文件，在编译过程中实现**任意代码执行**。

## 漏洞根因分析

### 1. 环境变量读取点 (SEC-001)

**文件**: `bishengir/lib/Tools/Utils/Utils.cpp`  
**行号**: 285-300  
**函数**: `getBiShengInstallPath()`

```cpp
std::string bishengir::getBiShengInstallPath() {
  const char *kBiShengInstallPathEnv = "BISHENG_INSTALL_PATH";
  const char *kBiShengInstallPath = getenv(kBiShengInstallPathEnv);  // [SOURCE] 无验证读取
  if (!kBiShengInstallPath) {
    LLVM_DEBUG(llvm::dbgs() << "[DEBUG] BISHENG_INSTALL_PATH is not set.\n");
    return "";
  }

  llvm::SmallString<128> path;
  path.append(kBiShengInstallPath);  // 直接使用攻击者控制的值
  std::error_code errorCode = llvm::sys::fs::make_absolute(path);
  if (errorCode)
    llvm::report_fatal_error("Failed to get absolute path for " + path);
  return path.str().str();  // [SINK] 返回给所有执行函数
}
```

**问题**：
- `getenv()` 直接读取环境变量，无任何验证
- 无路径白名单检查
- 无签名校验
- 无checksum验证
- 调用者无法区分合法路径与恶意路径

### 2. 二进制执行点 (SEC-002)

**文件**: `bishengir/lib/Tools/Utils/Utils.cpp`  
**行号**: 175-221  
**函数**: `execute()`

```cpp
LogicalResult bishengir::execute(StringRef binName, StringRef installPath,
                                  SmallVectorImpl<StringRef> &arguments,
                                  std::optional<llvm::StringRef> outputFile,
                                  unsigned timeoutSeconds) {
  std::string binPath;
  if (!installPath.empty()) {
    // [CRITICAL] 在攻击者控制的目录中搜索二进制
    if (auto binPathOrErr =
            llvm::sys::findProgramByName(binName, {installPath})) {
      binPath = binPathOrErr.get();  // 找到恶意二进制
    }
  }
  // ...
  arguments[0] = binPath;
  
  // [EXECUTION] 以当前进程权限执行恶意二进制
  if (llvm::sys::ExecuteAndWait(binPath, arguments, /*Env=*/std::nullopt,
                                 /*Redirects=*/redirects) != 0) {
    return failure();
  }
  return success();
}
```

**问题**：
- `findProgramByName()` 在 `installPath` 目录中搜索二进制
- 如果 `installPath` 是攻击者控制的路径，将找到伪造的二进制
- `ExecuteAndWait()` 直接执行，无任何校验

### 3. 调用入口点

所有以下入口点都调用 `execute(getBiShengInstallPath())`：

| 入口点 | 文件 | 行号 | 触发条件 |
|--------|------|------|----------|

| **runBishengirCompile91095** | bishengir-compile.cpp | 66-75 | `--target=Ascend910_95*` |
| **runBishengirOptA5** | bishengir-opt.cpp | 76-89 | `--target=Ascend910_95*` |
| **detectHIVMCVersion** | Utils.cpp | 254-283 | 版本检测（自动触发） |

## 攻击场景分析

### 场景 1: CI/CD 环境污染

**攻击向量**：攻击者控制 CI/CD 环境变量配置

```yaml
# 恶意 CI 配置示例
env:
  BISHENG_INSTALL_PATH: "/tmp/attacker-controlled"
```

**步骤**：
1. 攻击者在 CI 系统中植入恶意环境变量配置
2. 攻击者在其控制的服务器上放置伪造的 `hivmc`
3. CI 构建触发时，`bishengir-compile` 执行伪造二进制
4. 伪造二进制：
   - 收集构建环境敏感信息（API密钥、凭证）
   - 在产物中植入后门
   - 上传数据到攻击者服务器

**影响**：供应链攻击，影响所有下游用户

### 场景 2: 共享开发环境

**攻击向量**：多用户共享服务器，攻击者修改全局环境变量

```bash
# 攻击者操作
export BISHENG_INSTALL_PATH=/home/attacker/malicious
mkdir -p /home/attacker/malicious/bin
cp malicious_hivmc /home/attacker/malicious/bin/hivmc
chmod +x /home/attacker/malicious/bin/hivmc
```

**步骤**：
1. 攻击者在共享服务器上设置环境变量（通过 `.bashrc`, `/etc/profile.d/`, 或环境模块）
2. 其他用户运行编译时，无意中执行恶意二进制
3. 恶意代码以其他用户权限运行，可访问其数据和凭证

### 场景 3: Docker/Kubernetes 容器环境

**攻击向量**：容器镜像或 Pod 配置中的环境变量注入

```yaml
# 恶意 Pod 配置
apiVersion: v1
kind: Pod
spec:
  containers:
  - name: compiler
    env:
    - name: BISHENG_INSTALL_PATH
      value: "/malicious-path"
```

**影响**：容器内任意代码执行，可能突破容器隔离

### 场景 4: IDE/工具链集成

**攻击向量**：IDE 配置文件或工具链设置脚本

**步骤**：
1. 攻击者修改 IDE 配置（如 VSCode settings.json）
2. 配置中植入 `BISHENG_INSTALL_PATH`
3. 开发者使用 IDE 编译时触发漏洞

## 影响评估

### 直接影响

1. **任意代码执行**：攻击者可在编译过程中执行任意代码
2. **权限提升**：恶意代码以编译进程权限运行（可能是 root 或服务账户）
3. **数据窃取**：窃取构建环境中的敏感信息
4. **供应链污染**：在编译产物中植入后门

### 间接影响

1. **横向移动**：利用窃取的凭证攻击其他系统
2. **持久化**：植入长期后门
3. **信任链破坏**：损害编译器可信度

## 修复建议

### 1. 立即修复（高优先级）

#### 方案 A: 环境变量白名单验证

```cpp
std::string bishengir::getBiShengInstallPath() {
  const char *kBiShengInstallPathEnv = "BISHENG_INSTALL_PATH";
  const char *kBiShengInstallPath = getenv(kBiShengInstallPathEnv);
  if (!kBiShengInstallPath) {
    return "";
  }

  // [NEW] 白名单验证
  static const std::vector<std::string> allowedPrefixes = {
    "/usr/local/bishengir",
    "/opt/bishengir",
    "/usr/lib/bishengir",
    // 系统管理员可配置白名单
  };
  
  llvm::SmallString<128> path;
  path.append(kBiShengInstallPath);
  std::error_code errorCode = llvm::sys::fs::make_absolute(path);
  if (errorCode)
    return "";
  
  // 验证路径是否在白名单中
  bool isAllowed = false;
  for (const auto& prefix : allowedPrefixes) {
    if (path.str().startswith(prefix)) {
      isAllowed = true;
      break;
    }
  }
  
  if (!isAllowed) {
    llvm::errs() << "[SECURITY] BISHENG_INSTALL_PATH '" << path 
                 << "' is not in allowed paths. Ignoring.\n";
    return "";
  }
  
  return path.str().str();
}
```

#### 方案 B: 二进制签名校验

```cpp
LogicalResult bishengir::execute(StringRef binName, StringRef installPath,
                                  ...) {
  std::string binPath;
  if (!installPath.empty()) {
    if (auto binPathOrErr = llvm::sys::findProgramByName(binName, {installPath})) {
      binPath = binPathOrErr.get();
      
      // [NEW] 签名校验
      if (!verifyBinarySignature(binPath)) {
        llvm::errs() << "[SECURITY] Binary '" << binPath 
                     << "' signature verification failed.\n";
        return failure();
      }
    }
  }
  // ...
}

bool verifyBinarySignature(const std::string& binPath) {
  // 使用 GPG 或自定义签名验证
  // 1. 检查二进制签名文件 (binPath.sig)
  // 2. 验证签名与已知公钥匹配
  // 3. 返回验证结果
  return checkGPGSignature(binPath, "/usr/share/bishengir/keys/trusted.key");
}
```

#### 方案 C: 使用编译时固定路径

```cpp
// 编译时硬编码安装路径（最安全）
constexpr const char* K_DEFAULT_INSTALL_PATH = "/usr/lib/bishengir";

std::string bishengir::getBiShengInstallPath() {
  // 仅在特定条件下允许环境变量覆盖
  const char* envPath = getenv("BISHENG_INSTALL_PATH");
  if (!envPath) {
    return K_DEFAULT_INSTALL_PATH;
  }
  
  // 严格验证：必须是绝对路径，且属于可信目录
  if (!isPathUnderTrustedDirectory(envPath)) {
    llvm::errs() << "[SECURITY] Rejecting untrusted BISHENG_INSTALL_PATH\n";
    return K_DEFAULT_INSTALL_PATH;  // 回退到安全默认值
  }
  
  return std::string(envPath);
}
```
