# SEC-006: Python PATH 环境变量劫持漏洞

## 漏洞概览

| 属性 | 值 |
|------|-----|
| **漏洞ID** | SEC-006 |
| **CWE分类** | CWE-426: Untrusted Search Path |
| **严重性** | Medium → Verified: **High** |
| **置信度** | 70 (CONFIRMED) |
| **影响模块** | PythonWheel |
| **CVSS 3.1估算** | 6.5 (Medium) - AV:L/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:N |

## 漏洞描述

Python wheel 包 `ascendnpuir` 在 `_get_compiler_path()` 函数中使用 `subprocess.run(['which', 'bishengir-compile'])` 通过 PATH 环境变量查找编译器二进制。如果 PATH 被攻击者控制，`which` 命令将返回恶意路径，导致后续编译过程执行伪造的 `bishengir-compile` 二进制，实现**任意代码执行**。

## 漏洞根因分析

### 1. 源代码位置

**文件**: `bishengir/python/wheel/ascendnpuir/compiler.py`  
**行号**: 42-83  
**函数**: `_get_compiler_path()`

```python
def _get_compiler_path() -> Path:
    """Get the path to the bishengir-compile binary."""
    # Get the directory where this package is installed
    package_dir = Path(__file__).parent
    
    # Binary name - only need bishengir-compile
    binary_name = "bishengir-compile"
    
    # Look for the binary in the package's bin directory (new structure)
    binary_path = package_dir / "bin" / binary_name
    if binary_path.exists():
        return binary_path
    
    # Fallback to looking directly in the package directory (old structure)
    binary_path = package_dir / binary_name
    if binary_path.exists():
        return binary_path
    
    # [VULNERABLE] If not found in package, try to find in PATH
    try:
        result = subprocess.run(
            ["which", "bishengir-compile"],  # [SOURCE] PATH查找，无验证
            capture_output=True,
            text=True,
            check=False
        )
        if result.returncode == 0:
            # [SINK] 直接使用 which 返回的路径，无验证
            return Path(result.stdout.strip().split('\n')[0])
    except Exception:
        pass
    
    raise FileNotFoundError(
        f"Could not find bishengir-compile binary. "
        f"Expected location: {binary_path}"
    )
```

## 攻击场景分析

### 场景 1: Python 环境污染

**攻击向量**：修改 Python 虚拟环境的 PATH

```bash
# 攻击者在激活脚本中植入恶意 PATH
cat > /path/to/venv/bin/activate << 'EOF'
# 原始激活内容...
export PATH=/tmp/attacker:$PATH
EOF

# 用户激活虚拟环境
source /path/to/venv/bin/activate

# 用户编译 MLIR
python -c "import ascendnpuir; ascendnpuir.compile(mlir_code)"
# → 执行 /tmp/attacker/bishengir-compile
```

### 场景 2: 共享 Jupyter Notebook 环境

**攻击向量**：在 Jupyter kernel 配置中修改环境变量

```python
# 恶意 Notebook Cell
import os
os.environ['PATH'] = '/tmp/attacker:' + os.environ['PATH']

# 其他用户在同一 kernel 中编译
import ascendnpuir
ascendnpuir.compile(mlir_code)  # → 执行恶意二进制
```

### 场景 3: CI/CD Python 构建

**攻击向量**：CI 配置中的环境变量注入

```yaml
# 恶意 GitHub Actions 配置
steps:
  - name: Setup environment
    run: echo "PATH=/tmp/attacker:$PATH" >> $GITHUB_ENV
  
  - name: Build with ascendnpuir
    run: python build.py  # 触发恶意二进制执行
```

### 场景 4: 用户级 PATH 修改

**攻击向量**：修改 `.bashrc` 或 shell 配置

```bash
# 攻击者植入（如果可修改用户配置）
echo 'export PATH=/tmp/attacker:$PATH' >> ~/.bashrc
source ~/.bashrc

# 用户使用 Python API
python3 -c "import ascendnpuir; ascendnpuir.compile('...')"
```

```python
    """Get the path to the bishengir-compile binary."""

def verify_binary_safety(path: Path) -> bool:
    """Verify binary is not in suspicious locations."""
    # 检查路径是否在可信目录中
    trusted_prefixes = [
        "/usr/lib/bishengir",
        "/usr/local/lib/bishengir",
        "/opt/bishengir",
        # 允许包内安装
    ]
    
    abs_path = path.resolve()
    
    # 检查是否在临时目录或可疑路径
    suspicious_prefixes = [
        "/tmp",
        "/var/tmp",
        "/home",
        getenv("HOME", ""),
    ]
    
    for prefix in suspicious_prefixes:
        if str(abs_path).startswith(prefix):
            return False
    
    # 检查所有权和权限
    stat = path.stat()
    if stat.st_mode & 0o002:  # 其他用户可写
        return False
    
    return True
```
