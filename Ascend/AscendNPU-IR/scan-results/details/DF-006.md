## 1. 漏洞根因分析

### 1.1 漏洞背景：版本检测机制

BiShengIR 编译器在执行编译管线前需要检测 HIVMC（华为 IR 虚拟机编译器）的版本，以确保向后兼容性。这个版本检测功能通过执行外部 `hivmc --version` 命令实现，但存在严重的安全缺陷。

### 1.2 漏洞代码详解

**位置**: `bishengir/lib/Tools/Utils/Utils.cpp:254-283`

```cpp
std::optional<llvm::VersionTuple>
bishengir::detectHIVMCVersion(StringRef hivmcName) {

  TempDirectoriesStore tempDirsStore;
  auto fileHandler = getTempFile("version.out", tempDirsStore);
  if (!fileHandler) {
    llvm::dbgs() << "[ERROR] Failed to create temporary input file needed to "
                    "detect hivmc version.\n";
    return std::nullopt;
  }
  std::string fileName = fileHandler->outputFilename();
  fileHandler->os().flush();

  SmallVector<StringRef> args{"", "--version"};
  StringRef outputFile = fileHandler->getFilename();
  
  // [漏洞核心] 使用未经信任的安装路径执行外部二进制
  if (failed(execute(hivmcName, getBiShengInstallPath(), args, outputFile))) {
    llvm::dbgs() << "[ERROR] Failed to run `hivmc --version`.\n";
    return std::nullopt;
  }

  std::string errorMsg;
  std::unique_ptr<llvm::MemoryBuffer> buffer =
      mlir::openInputFile(fileName, &errorMsg);
  if (!buffer) {
    llvm::dbgs() << "[ERROR] Failed to open file: " << fileName << "\n";
    return std::nullopt;
  }

  return findHIVMCVersion(buffer->getBuffer());
}
```

### 1.4 execute() 函数详解

```cpp
LogicalResult bishengir::execute(StringRef binName, StringRef installPath,
                                 SmallVectorImpl<StringRef> &arguments,
                                 std::optional<llvm::StringRef> outputFile,
                                 unsigned timeoutSeconds) {
  std::string binPath;
  
  // [关键漏洞点] 优先在 installPath 中搜索二进制
  if (!installPath.empty()) {
    if (auto binPathOrErr =
            llvm::sys::findProgramByName(binName, {installPath})) {
      binPath = binPathOrErr.get();  // ← 攻击者控制的路径被使用
    } else {
      llvm::errs() << "[WARNING] Cannot find " << binName << " under "
                   << installPath << "\n";
    }
  }
  
  // [降级搜索] 如果 installPath 下没找到，才搜索系统 PATH
  if (binPath.empty()) {
    if (auto binPathOrErr = llvm::sys::findProgramByName(binName)) {
      binPath = binPathOrErr.get();
    } else {
      llvm::errs() << "[ERROR] Cannot find " << binName << " under $PATH\n";
      return failure();
    }
  }
  
  arguments[0] = binPath;
  
  // [漏洞触发] 执行攻击者控制的二进制
  SmallVector<std::optional<StringRef>> redirects = {
      /*stdin=*/std::nullopt, /*stdout=*/outputFile, /*stderr=*/std::nullopt};
  
  if (llvm::sys::ExecuteAndWait(binPath, arguments, /*Env=*/std::nullopt,
                                /*Redirects=*/redirects) != 0) {
    llvm::errs() << "[ERROR] Executing: ";
    llvm::interleave(arguments, llvm::errs(),
                     [](const StringRef &arg) { llvm::errs() << arg; }, " ");
    llvm::errs() << "\n";
    return failure();
  }
  return success();
}
```

**向量字符串**: CVSS:3.1/AV:L/AC:L/PR:L/UI:N/S:U/C:H/I:H/A:L

### 4.5 特殊风险点

1. **版本号伪造**: 攻击者可以返回任意版本号，可能触发不安全的兼容性代码路径
2. **隐蔽执行**: `--version` 调用预期是只读操作，用户不会怀疑代码执行
3. **调试信息泄露**: 版本检测可能输出调试信息，泄露系统配置

### 5.3 防御深度措施

#### 1. 二进制完整性验证

```cpp

      }
    }
  }

}

bool isSecureInstallPath(const std::string& path) {
  // 1. 路径解析（解析所有符号链接）
  llvm::SmallString<256> realPath;
  if (llvm::sys::fs::real_path(path, realPath)) {
    return false;
  }
  
  // 2. 路径所有权检查
  struct stat st;
  if (stat(realPath.c_str(), &st) != 0) {
    return false;
  }
  
  // 目录必须由 root 或当前用户所有
  if (st.st_uid != 0 && st.st_uid != getuid()) {
    return false;
  }
  
  // 3. 目录权限检查（不能全局可写）
  if ((st.st_mode & S_IWOTH) != 0) {
    return false;  // 其他用户可写
  }
  
  // 4. 路径逃逸检查
  if (realPath.str().contains("..")) {
    return false;
  }
  
  // 5. 路径白名单检查
  static const std::vector<std::string> allowedPrefixes = {
    "/usr/local/bishengir",
    "/opt/bishengir",
    "/usr/lib/bishengir",
  };
  
  std::string pathStr = realPath.str().str();
  for (const auto& prefix : allowedPrefixes) {
    if (pathStr.find(prefix) == 0) {
      return true;
    }
  }
  
  return false;
}
```

#### 2. 版本输出验证

```cpp
llvm::VersionTuple findHIVMCVersion(llvm::StringRef content) {
  StringRef versionLine = content.split('\n').first;
  
  // [新增] 内容长度限制（防止缓冲攻击）
  if (versionLine.size() > 256) {
    llvm::dbgs() << "[WARNING] Suspiciously long version output\n";
    return llvm::VersionTuple();
  }
  
  // [新增] 字符白名单检查
  for (char c : versionLine) {
    if (!isAllowedVersionChar(c)) {
      llvm::dbgs() << "[WARNING] Invalid character in version output\n";
      return llvm::VersionTuple();
    }
  }
  
  Regex R("^(hivmc) "
          "([0-9]+\\.[0-9]+\\.[0-9]+) "
          "\\(([0-9a-fA-F]{6,40}) "
          "([0-9]{4}-[0-9]{2}-[0-9]{2})\\)$");
  
  SmallVector<StringRef, 4> M;
  if (!R.match(versionLine, &M)) {
    return llvm::VersionTuple();
  }

  // [新增] 版本号范围验证
  StringRef versionStr = M[2];
  auto version = bishengir::parseHIVMCVersion(versionStr);
  if (version.has_value()) {
    // 验证版本号在合理范围内
    if (version->getMajor() > 100 || version->getMinor() > 100) {
      llvm::dbgs() << "[WARNING] Suspicious version number: " 
                   << versionStr << "\n";
      return llvm::VersionTuple();
    }
    return version.value();
  }

  return llvm::VersionTuple();
}
```

## 6. 总结

### 6.1 漏洞关键点

1. **数据源**: `BISHENG_INSTALL_PATH` 环境变量完全由攻击者控制
2. **传播路径**: `getBiShengInstallPath()` → `detectHIVMCVersion()` → `execute()` → `findProgramByName()`
3. **攻击点**: 在攻击者控制的路径中执行伪造的 `hivmc` 二进制
4. **影响**: 任意代码执行 + 版本号伪造

### 6.2 与其他漏洞的关系

DF-006 是 SEC-001 系列漏洞的一个特定实例，专门针对版本检测功能。虽然触发条件有限（仅编译开始时一次），但仍然是完整攻击链的一部分：

```
SEC-001 (源头) ─┬─→ DF-006 (版本检测劫持)
               │
               ├─→ SEC-002 (编译过程劫持)
               │
               └─→ SEC-003 (其他外部工具调用)
```

### 6.3 修复建议总结

**最优先**: 在 `getBiShengInstallPath()` 中添加路径安全验证
**长期**: 移除对环境变量的依赖，使用编译时配置或安全配置文件
