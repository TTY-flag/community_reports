# VULN-PWC-005-2026: Security Control Inconsistency (Read Path Validation Missing)

## 漏洞摘要

| 属性 | 值 |
|------|-----|
| ID | VULN-PWC-005-2026 |
| 类型 | Inconsistent Security Controls |
| CWE | CWE-693 (Incomplete Blacklist) |
| 严重性 | Medium |
| 置信度 | 80 |
| 状态 | CONFIRMED |
| 关联漏洞 | VULN-DF-CPP-001 (Critical) - 同一缺陷的攻击可行性视角 |

## 漏洞描述

**架构缺陷**：系统实现了针对**写操作**的完整安全验证机制（`File::CheckFileBeforeCreateOrWrite()`），但**读操作**完全没有任何安全验证。这种不对称的安全设计暴露了系统架构层面的设计偏差：

- **写操作**（WriteDataToFile）：调用 `File::CheckFileBeforeCreateOrWrite(filePath, true)` 进行严格验证
- **读操作**（GetDataFromBin）：直接使用 `std::ifstream` 打开文件，**零验证**

### 安全控制对比表

| 检查项 | WriteDataToFile | GetDataFromBin |
|--------|-----------------|----------------|
| 路径规范化（处理 `..`） | ✓ GetAbsPath() | ❌ 无 |
| 路径长度验证 | ✓ ≤4096字符 | ❌ 无 |
| 路径字符验证 | ✓ 正则匹配 | ❌ 无 |
| 路径深度验证 | ✓ ≤32层 | ❌ 无 |
| 符号链接检查 | ✓ 不允许软链接 | ❌ 无 |
| 文件所有权检查 | ✓ 必须是当前用户 | ❌ 无 |
| 文件权限检查 | ✓ 权限≤750 | ❌ 无 |
| 父目录验证 | ✓ CheckDir() | ❌ 无 |

## 架构缺陷分析

### 为什么会出现这种不对称？

**设计偏差假设**：
1. 开发团队可能认为"写入"是高风险操作（数据篡改），而"读取"是低风险
2. File.h 头文件中定义了 `CheckFileBeforeCreateOrWrite()`，注释明确为"写文件前的校验"
3. **没有定义 `CheckFileBeforeRead()` 类似的函数**，表明设计时未考虑读操作的安全性

### 为什么读操作也需要验证？

读操作的安全风险：
1. **信息泄露**：读取敏感配置文件、密钥、凭证
2. **路径遍历**：使用 `../../../etc/passwd` 绕过目录限制
3. **符号链接攻击**：通过符号链接读取受限文件
4. **跨用户读取**：读取其他用户的私有文件

### File::CheckFileBeforeCreateOrWrite 的验证项详解

```cpp
// File.cpp:256-299
bool File::CheckFileBeforeCreateOrWrite(const std::string &path, bool overwrite)
{
    std::string absPath = GetAbsPath(path);           // 1. 路径规范化
    if (!IsPathLengthLegal(absPath)) return false;    // 2. 长度≤4096
    if (!IsPathCharactersValid(absPath)) return false;// 3. 字符白名单 [a-zA-Z0-9_./-]
    if (!IsPathDepthValid(absPath)) return false;     // 4. 深度≤32
    if (IsPathExist(absPath)) {
        if (!IsRegularFile(absPath)) return false;    // 5. 必须是普通文件
        if (IsSoftLink(absPath)) return false;        // 6. 不允许软链接
        if (!IsFileWritable(absPath)) return false;   // 7. 权限检查
        if (!CheckOwner(absPath)) return false;       // 8. 所有权检查
    }
    return CheckDir(GetParentDir(absPath));           // 9. 父目录验证
}
```

**GetDataFromBin 完全绕过以上所有检查**。

## 攻击链验证

### 攻击路径

```
[外部输入] argv[8] (inputWeightPath) ─ 完全可控
    │
    ▼
[数据传递] main.cpp:224 const string inputWeightPath = argv[8]
    │
    ▼
[到达漏洞点] main.cpp:260 GetDataFromBin(inputWeightPath, ...)
    │
    ▼
[触发漏洞] graph_utils.cpp:41 std::ifstream inFile(input_path, ios::binary)
    │
    ▼
[无安全阻断] 直接打开任意路径文件
```

### 限制条件分析

GetDataFromBin 中存在文件大小验证：
```cpp
// graph_utils.cpp:55-58
uint64_t dataLen = static_cast<uint64_t>(size) * data_type_size;  // size = dimK * dimN * 16 * 32
if (dataLen != static_cast<uint64_t>(fileSize)) {
    std::cout << "Invalid param: expected len=" << dataLen << ", but file size=" << fileSize << std::endl;
    return false;
}
```

**关键**：`dimK` 和 `dimN` 来自 `argv[1]` 和 `argv[2]`，**完全由攻击者控制**。

### 绕过大小限制的方法

攻击者可以：
1. 获取目标文件大小：`stat /etc/passwd` 或 `ls -l`
2. 计算所需的 dimK 和 dimN：`fileSize = dimK * dimN * 16 * 32`
3. 设置合适的参数值（需满足 CheckShape 的 `val ≤ 50000` 限制）

**最大可读取文件大小**：`50000 * 50000 * 16 * 32 = 40GB`，覆盖几乎所有敏感文件。

### PoC 构造示例

```bash

# 计算：2000 = dimK * dimN * 16 * 32

# 简化 PoC 思路：

# 2. 计算精确的 dimK * dimN = fileSize / (16 * 32)
# 3. 执行程序

# dimK=1, dimN=2 → size=1024 字节，匹配目标
```

### 互补价值

VULN-PWC-005-2026 揭示了**为什么**会存在 VULN-DF-CPP-001：
- 系统设计时建立了完整的安全验证框架（File 类）
- 但该框架**只覆盖写操作**，未考虑读操作
- 这反映了设计思维的偏差：认为"写入危险，读取安全"

两个漏洞报告应合并考虑：
- VULN-DF-CPP-001 定义了**立即修复**的优先级（Critical）
- VULN-PWC-005-2026 定义了**架构改进**的方向（建立对称安全模型）

## 修复建议

### 短期修复（立即）

为 GetDataFromBin 添加安全验证：

```cpp
// graph_utils.cpp:39-45
bool GetDataFromBin(std::string input_path, std::vector<int64_t> shapes, uint8_t *&data, int data_type_size)
{
    // 添加读操作安全验证
    if (!File::CheckFileBeforeRead(input_path)) {
        std::cout << "Security check failed for: " << input_path << std::endl;
        return false;
    }
    
    std::ifstream inFile(input_path, std::ios::binary);
    ...
}
```

### 长期修复（架构改进）

1. **添加 File::CheckFileBeforeRead() 函数**：

```cpp
// File.h - 新增
static bool CheckFileBeforeRead(const std::string &path);

// File.cpp - 新增实现
bool File::CheckFileBeforeRead(const std::string &path)
{
    std::string absPath = GetAbsPath(path);
    if (absPath.empty()) return false;
    if (!IsPathLengthLegal(absPath)) return false;
    if (!IsPathCharactersValid(absPath)) return false;
    if (!IsPathDepthValid(absPath)) return false;
    if (!IsPathExist(absPath)) return false;
    if (!IsRegularFile(absPath)) return false;
    if (IsSoftLink(absPath)) return false;         // 阻止符号链接攻击
    // 读操作不需要所有权检查，但应验证文件是否可读
    return CheckDir(GetParentDir(absPath));
}
```

2. **建立对称安全模型**：

| 操作类型 | 验证函数 | 验证项 |
|---------|---------|--------|
| 读文件 | CheckFileBeforeRead | 路径规范、字符验证、软链接检查、权限检查 |
| 写文件 | CheckFileBeforeCreateOrWrite | 所有读检查 + 所有权检查 + 写权限检查 |
| 创建目录 | CheckDir | 路径规范、权限、所有权 |

3. **代码审查清单**：
- 检查所有文件操作是否有对应的安全验证
- 确保 File.h 提供的验证函数覆盖所有操作类型
- 将安全验证作为强制性代码审查项
