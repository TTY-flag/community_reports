# SEC-003：AutoModel kwargs传递trust_remote_code致远程代码执行

## 漏洞概述

**漏洞 ID**: SEC-003  
**类型**: Remote Code Execution (CWE-94)  
**严重程度**: Critical  
**置信度**: 80 (CONFIRMED)  
**主报告关联**: SEC-011 (最终执行点)

---

## 漏洞定位

**文件**: `tensor_cast/transformers/utils.py`  
**行号**: 334-338  
**函数**: `AutoModelConfigLoader.try_to_load_model`

### 漏洞代码

```python
@staticmethod
def try_to_load_model(*args, remote_source=..., **kwarg):
    ...
    try:
        hf_model = AutoModel.from_config(*args, **kwarg)
    except Exception:
        hf_model = AutoModelForCausalLM.from_config(*args, **kwarg)
    return hf_model
```

---

## 在漏洞链中的位置

```
SEC-011 → SEC-012 → SEC-001 → SEC-002 → SEC-003 → RCE
                                        ↓
                                   kwargs 执行点
```

**角色**: HuggingFace transformers 库的最终执行接口

---

## 漏洞机制

| kwargs 内容 | 行为 |
|-------------|------|
| `trust_remote_code=True` | 执行模型仓库中的自定义代码 |
| `trust_remote_code=False` | 仅使用原生代码 |

**关键问题**:
- kwargs 直接传递，无安全检查
- 二次回退机制 (`AutoModel` → `AutoModelForCausalLM`)
- 执行来自远程仓库的 Python 文件

---

## 执行流程

```
try_to_load_model(**kwarg)
    → kwarg 含 trust_remote_code=True
    → AutoModel.from_config(trust_remote_code=True)
    → transformers 库下载 configuration_xxx.py
    → Python import 执行
    → RCE
```

---

## 修复建议

参见主报告 SEC-011.md 第 5.2 节。

**核心修复**:
- 在 kwargs 传递前提取并验证 `trust_remote_code`
- 添加显式警告和确认机制
- 限制异常捕获范围
