# SEC-012: ConfigResolver 自动启用 RCE

## 漏洞概述

**漏洞 ID**: SEC-012  
**类型**: Remote Code Execution (CWE-94)  
**严重程度**: Critical  
**置信度**: 85 (CONFIRMED)  
**主报告关联**: SEC-011 (Layer 2 触发点)

---

## 漏洞定位

**文件**: `tensor_cast/core/config_resolver.py`  
**行号**: 83-85  
**函数**: `ConfigResolver.__init__`

### 漏洞代码

```python
# 根据原生支持状态自动设置 trust_remote_code
self.model_config.trust_remote_code = (
    not auto_loader.is_transformers_natively_supported
)
```

---

## 在漏洞链中的位置

```
SEC-011 (根因) ─→ SEC-012 (Layer 2) ─→ SEC-001 (Layer 3)
                    ↓
               非原生模型 → 自动 True
                    ↓
               RCE 执行
```

**角色**: Layer 2 - 非原生模型自动启用机制

---

## 漏洞机制

| 检测结果 | 行为 |
|----------|------|
| `is_transformers_natively_supported = True` | `trust_remote_code = False` |
| `is_transformers_natively_supported = False` | `trust_remote_code = True` ← 危险 |

**关键问题**:
- 任何不在 transformers 原生列表中的模型都会自动启用 RCE
- 无用户确认
- 无安全警告
- 与 SEC-001 形成双重保障

---

## 受影响场景

| 模型类型 | 触发 | 说明 |
|----------|------|------|
| 官方 transformers 模型 | 不触发 | 原生支持 |
| 第三方开源模型 | 触发 | 需自定义配置 |
| 企业私有模型 | 触发 | 自定义架构 |
| 恶意构造模型 | 触发 | 攻击者设计 |

---

## 数据流追踪

```
ConfigResolver.__init__(model_id)
    → AutoModelConfigLoader.load_config()
    → is_transformers_natively_supported = False
    → self.model_config.trust_remote_code = True
    → TransformerModel.__init__ → 传递 trust_remote_code=True
    → RCE
```

---

## 修复建议

参见主报告 SEC-011.md 第 5.1 节 (修复 3)。

**核心修复**: 移除自动设置逻辑，完全依赖用户显式配置。
