# SEC-001: AutoConfig 自动回退触发 RCE

## 漏洞概述

**漏洞 ID**: SEC-001  
**类型**: Remote Code Execution (CWE-94)  
**严重程度**: Critical  
**置信度**: 85 (CONFIRMED)  
**主报告关联**: SEC-011 (Layer 3 触发点)

---

## 漏洞定位

**文件**: `tensor_cast/transformers/utils.py`  
**行号**: 258-260  
**函数**: `AutoModelConfigLoader.load_config`

### 漏洞代码

```python
# 尝试原生加载，失败时自动启用 trust_remote_code=True
try:
    hf_config = AutoConfig.from_pretrained(model_id)
    self.is_transformers_natively_supported = True
except Exception:
    hf_config = AutoConfig.from_pretrained(model_id, trust_remote_code=True)
```

---

## 在漏洞链中的位置

```
SEC-011 (根因) ─→ SEC-012 (Layer 2) ─→ SEC-001 (Layer 3 回退)
                                        ↓
                                    RCE 执行
```

**角色**: Layer 3 - 加载失败自动回退触发点

---

## 漏洞机制

| 触发条件 | 触发路径 |
|----------|----------|
| 模型类型不存在 | `AutoConfig.from_pretrained()` 抛出异常 |
| config.json 格式错误 | 异常捕获 → 自动 `trust_remote_code=True` |
| 网络问题 | 任何 `Exception` → 启用远程代码执行 |

**关键问题**:
- 捕获 **所有异常类型** (`Exception`)
- 无用户确认机制
- 无安全警告日志
- 无条件启用危险参数

---

## 数据流追踪

```
CLI args.model_id
    → UserInputConfig.model_id
    → ConfigResolver.__init__
    → AutoModelConfigLoader.load_config(model_id)
    → AutoConfig.from_pretrained(model_id, trust_remote_code=True)
    → 执行 configuration_xxx.py
    → RCE
```

---

## 攻击利用

攻击者可故意构造触发加载失败的模型仓库：

```
恶意仓库结构:
├── config.json (含无效 model_type 或错误格式)
├── configuration_malicious.py (恶意代码)
└── model.safetensors

触发流程:
→ AutoConfig.from_pretrained() → 解析失败 → Exception
→ 自动回退 → trust_remote_code=True
→ 下载并执行 configuration_malicious.py
→ RCE
```

---

## 修复建议

参见主报告 SEC-011.md 第 5.1 节 (修复 2)。

**核心修复**: 移除自动回退机制，要求用户显式确认。
