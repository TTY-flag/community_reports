# SEC-011：trust_remote_code漏洞链致多层级远程代码执行

## 漏洞链概述

**主漏洞 ID**: SEC-011  
**关联漏洞**: SEC-001, DF-001, SEC-012, SEC-002, SEC-003, DF-002, DF-012  
**漏洞类型**: Remote Code Execution (CWE-94)  
**严重程度**: Critical  
**置信度**: 85-90

本报告分析 MindStudio-Modeling 项目中一个严重的远程代码执行漏洞链，该漏洞链源于 `trust_remote_code=True` 的默认配置和自动回退机制，允许攻击者通过恶意模型仓库执行任意 Python 代码。

---

## 1. 漏洞链架构分析

### 1.1 核心设计缺陷

整个漏洞链由**三个相互协同的设计缺陷**构成：

| 层级 | 漏洞点 | 代码位置 | 影响机制 |
|------|--------|----------|----------|
| **第一层 (Root Cause)** | ModelConfig 默认值 | `model_config.py:448` | `trust_remote_code: bool = True` 默认启用 |
| **第二层 (Auto Enable)** | ConfigResolver 自动设置 | `config_resolver.py:83-85` | 非原生模型自动设置 `trust_remote_code=True` |
| **第三层 (Fallback)** | AutoConfig 加载回退 | `transformers/utils.py:258-260` | 加载失败时自动启用 `trust_remote_code=True` |

这三个层级形成**冗余攻击路径**，即使修复其中一个，其他层级仍可触发 RCE。

### 2.3 Layer 详解

#### Layer 1: ModelConfig 默认值 (SEC-011)

**位置**: `tensor_cast/model_config.py:448`

```python
@dataclasses.dataclass
class ModelConfig:
    ...
    trust_remote_code: bool = True  # ← 致命默认值
    remote_source: str = RemoteSource.huggingface
```

**分析**:
- 这是**根因缺陷**，默认启用远程代码执行
- 用户无需任何操作就处于危险状态
- 即使 CLI 不传递此参数，默认值仍然生效
- `ModelConfig` 是整个框架的配置核心，所有下游组件继承此设置

#### Layer 2: ConfigResolver 自动启用 (SEC-012)

**位置**: `tensor_cast/core/config_resolver.py:83-85`

```python
class ConfigResolver:
    def __init__(self, ...):
        ...
        self.model_config.trust_remote_code = (
            not auto_loader.is_transformers_natively_supported
        )
```

**分析**:
- 系统自动检测模型是否被 transformers 原生支持
- 对于**任何自定义模型**，自动设置 `trust_remote_code=True`
- 这意味着攻击者只需提供一个"非原生"模型即可触发 RCE
- 与 Layer 1 形成**双重保障**：即使修复默认值，此层级仍触发

#### Layer 3: 加载失败回退 (SEC-001, DF-001)

**位置**: `tensor_cast/transformers/utils.py:255-260`

```python
def load_config(self, model_id, remote_source=...):
    ...
    try:
        hf_config = AutoConfig.from_pretrained(model_id)
        self.is_transformers_natively_supported = True
    except Exception:
        hf_config = AutoConfig.from_pretrained(model_id, trust_remote_code=True)
```

**分析**:
- 这是最危险的触发路径
- **任何加载异常**都会启用 `trust_remote_code=True`
- 攻击者可故意构造加载失败的配置（如无效 model_type）
- 异常类型不限（`Exception` 捕获所有）
- 无日志警告、无用户确认、无安全检查

## 8. 总结

### 根因
`ModelConfig` 类将 `trust_remote_code=True` 设为默认值，形成漏洞链的根因缺陷。

### 危害
- **用户无感知**: 默认启用意味着用户无需任何操作就处于危险状态
- **多重触发**: 三层冗余机制确保即使修复一处，其他层级仍可触发
- **完整权限**: 恶意代码获得完整 Python 执行环境

### 修复核心
1. **立即**: 将所有默认值改为 `False`
2. **必须**: 移除自动回退机制
3. **建议**: 添加用户确认和安全验证
