# SEC-002: TransformerModel 参数传递执行点

## 漏洞概述

**漏洞 ID**: SEC-002  
**类型**: Remote Code Execution (CWE-94)  
**严重程度**: Critical  
**置信度**: 82 (CONFIRMED)  
**主报告关联**: SEC-011 (模型加载执行点)

---

## 漏洞定位

**文件**: `tensor_cast/transformers/model.py`  
**行号**: 185-189  
**函数**: `TransformerModel.__init__`

### 漏洞代码

```python
# 传递 trust_remote_code 参数到模型加载
self._inner = auto_loader.load_model(
    self.hf_config,
    self.model_config.dtype,
    trust_remote_code=self.model_config.trust_remote_code,
)
```

---

## 在漏洞链中的位置

```
SEC-011 → SEC-012 → SEC-001 → SEC-002 → RCE
                              ↓
                         参数传递执行点
```

**角色**: 模型加载阶段的最终参数传递点

---

## 漏洞机制

此漏洞是上游漏洞链的执行节点：

| 上游状态 | SEC-002 行为 |
|----------|--------------|
| SEC-011: 默认 True | 继承 → `trust_remote_code=True` |
| SEC-012: 自动 True | 继承 → `trust_remote_code=True` |
| SEC-001: 回退 True | 继承 → `trust_remote_code=True` |

**关键问题**:
- 无二次验证机制
- 无参数安全检查
- 直接传递危险参数

---

## 数据流追踪

```
ModelConfig.trust_remote_code (来自 SEC-011/012)
    → TransformerModel.__init__
    → model_config.trust_remote_code
    → auto_loader.load_model(trust_remote_code=True)
    → AutoModel.from_config(trust_remote_code=True)
    → RCE
```

---

## 修复建议

参见主报告 SEC-011.md 第 5.2 节。

**核心修复**: 
1. 在传递前添加验证检查
2. 添加用户确认机制
3. 记录安全审计日志
