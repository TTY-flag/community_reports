# 6c40d5f31c11：compile_executable不受信任构建脚本致任意代码执行

## 漏洞概述

| 属性 | 值 |
|------|-----|
| **漏洞ID** | 6c40d5f31c11 |
| **漏洞类型** | Untrusted Code Execution (关联 CWE-78: OS Command Injection, CWE-94: Code Injection) |
| **严重程度** | Critical |
| **置信度** | 90% → **已确认** |
| **影响版本** | MindStudio-Kernel-Launcher (当前版本) |
| **攻击复杂度** | 低 |
| **影响范围** | 任意代码执行 |
| **漏洞位置** | `mskl/launcher/compiler.py:369-370` |

### 漏洞摘要

MindStudio-Kernel-Launcher 的 `compile_executable()` 函数接收用户提供的 `build_script` 路径，并通过 `subprocess.run(['bash', build_script, ...])` 执行。虽然 `FileChecker` 对脚本文件进行了元数据验证（存在性、权限、大小等），但**完全不验证脚本内容**，导致攻击者可以提供恶意脚本并在目标环境执行任意命令。

此漏洞与已报告的 `compile()` 函数漏洞 (9b345463d9b3) 属于同一漏洞模式，但存在重要差异：

1. **不同的输出类型**: `compile_executable()` 编译可执行文件，而非共享库
2. **特殊的 prelaunch 模式**: 当 `context.prelaunch_flag=True` 时跳过输入检查
3. **与 autotune_v2 装饰器的关联**: 可通过装饰器间接触发

### 差异分析

| 差异点 | compile() | compile_executable() |
|--------|-----------|---------------------|
| **输出类型** | `.so` 共享库 | 可执行文件 (无扩展名要求) |
| **返回类型** | CompiledKernel | CompiledExecutable |
| **默认输出** | `_gen_module.so` | `_gen_executable` |
| **超时时间** | 600秒 | 120秒 |
| **prelaunch_flag** | 不影响 | **关键影响** |
| **调用场景** | 直接调用 / autotune | **autotune_v2 装饰器** |

### prelaunch_flag 的安全影响

`compile_executable()` 有一个特殊的安全机制：

```python
# compiler.py:353-367
if not context.prelaunch_flag:
    _check_compile_executable_input(build_script, src_file, output_bin_path, use_cache)

...

if context.prelaunch_flag:
    context.build_script = build_script  # 仅保存，不执行
    context.kernel_src_file = abs_src_path
    return CompiledExecutable(abs_output_bin_path)  # 返回空对象
```

当 `prelaunch_flag=True` 时：
- **跳过所有输入检查**（包括 FileChecker）
- **不执行 bash 命令**
- 仅保存 `build_script` 到 context 并返回空 CompiledExecutable

但这**不是安全缓解**，因为：
1. prelaunch 模式仅用于参数收集
2. 实际编译时 `prelaunch_flag=False`，仍会执行脚本
3. `build_script` 在 context 中保存后，后续编译时会使用

#### 1. autotune_v2 装饰器定义 - `tuner.py:557-578`

```python
def autotune_v2(configs: list, warmup_times: int = 5):
    """Decorator for auto-tuning an executable."""
    def decorator(func):
        def wrapper(*args, **kwargs):
            try:
                logger.debug('Starting autotune_v2...')
                autotune_utils.check_autotune_v2_params(configs, warmup_times)
                launch_params = get_params_from_pre_launch(func, *args, **kwargs)  # ← prelaunch 执行用户函数
                scheduler = AutotuneV2Scheduler(configs, warmup_times, launch_params)
                scheduler.execute()  # ← 实际编译执行
            except Exception as e:
                raise Exception(f'autotune_v2 failed') from e
        return wrapper
    return decorator
```

```python
def compile_executable(build_script: str,
                       src_file: str,
                       output_bin_path: str = "_gen_executable",
                       use_cache: bool = False) -> CompiledExecutable:

    def _check_compile_executable_input(build_script, launch_src_file, output_bin_path, use_cache):
        safe_check.check_variable_type(build_script, str)
        checker = FileChecker(build_script, "file")
        if not checker.check_input_file():
            raise Exception("Check the build_script {} permission failed.".format(build_script))
        # ... 其他检查

    if not context.prelaunch_flag:  # ← prelaunch 模式跳过检查
        _check_compile_executable_input(build_script, src_file, output_bin_path, use_cache)

    abs_src_path = os.path.realpath(src_file)
    abs_output_bin_path = os.path.realpath(output_bin_path)

    if use_cache and not context.autotune_in_progress:
        if not os.path.exists(abs_output_bin_path):
            raise Exception("The executable generated from build script does not exist.")
        return CompiledExecutable(abs_output_bin_path)

    if context.prelaunch_flag:  # ← prelaunch 模式：仅保存，不执行
        context.build_script = build_script
        context.kernel_src_file = abs_src_path
        return CompiledExecutable(abs_output_bin_path)

    # [SINK] 实际执行点
    compile_cmd = ["bash", build_script, abs_src_path, abs_output_bin_path]
    result = subprocess.run(compile_cmd, capture_output=True, text=True, timeout=120)  # ← 漏洞点！
    if result.returncode != 0:
        raise Exception("Compile failed.\nCommand info: " + ' '.join(compile_cmd) + "\n{}".format(result.stderr))

    return CompiledExecutable(abs_output_bin_path)
```

## FileChecker 局限性分析

`compile_executable()` 使用 `FileChecker` 进行输入验证，但其验证范围有限：

### 检查项列表

| 检查项 | 是否实现 | 是否阻止 | 安全效果 |
|--------|----------|----------|----------|
| 路径非法字符检查 | ✓ | ✓ | 防止路径注入，不影响内容 |
| 文件存在性检查 | ✓ | ✓ | 确保文件存在 |
| 路径长度限制 | ✓ | ✓ | 防止缓冲区溢出 |
| 软链接检查 | ✓ | ⚠ 仅警告 | **可被绕过** |
| 文件权限检查 | ✓ | ✓ | 防止权限绕过 |
| 文件大小限制 | ✓ | ✓ | 防止资源耗尽 |
| 文件所有者检查 | ✓ | ⚠ 仅警告 | **可被绕过** |
| 组/其他用户写权限检查 | ✓ | ⚠ 仅警告 | **可被绕过** |
| **文件内容检查** | ✗ | ✗ | **完全缺失** |
| **脚本命令白名单** | ✗ | ✗ | **完全缺失** |
| **危险模式检测** | ✗ | ✗ | **完全缺失** |

### prelaunch_flag 的影响

当 `context.prelaunch_flag=True` 时，`compile_executable()` **完全跳过 FileChecker 检查**：

```python
if not context.prelaunch_flag:
    _check_compile_executable_input(...)  # ← 跳过
```

这意味着在 autotune_v2 的 prelaunch 阶段，**任何脚本路径都会被接受并保存到 context**。

## 修复建议

### 1. 短期修复：内容验证（优先级：P0）

在 `FileChecker` 中添加脚本内容安全检查：

```python
import re

class FileChecker:
    DANGEROUS_PATTERNS = [
        r'\brm\s+-rf\b',
        r'\bchmod\s+[0-7]{3,4}\b',
        r'\bchown\b',
        r'\bcurl\s+.*\|\s*bash',
        r'\bwget\s+.*\|\s*bash',
        r'\bbash\s+-i\b',
        r'\bnc\s+-[elp]',
        r'/dev/tcp/',
        r'\bexport\b.*\bPATH\b',
        r'\beval\b',
        r'\bexec\b',
        r'\bsource\b.*http',
    ]
    
    def check_script_content(self):
        """检查脚本内容是否包含危险命令"""
        if self.file_type not in ["file", "app"]:
            return True
        
        try:
            with open(self.absolute_path, 'r', encoding='utf-8', errors='ignore') as f:
                content = f.read()
            
            for pattern in self.DANGEROUS_PATTERNS:
                if re.search(pattern, content, re.IGNORECASE):
                    logger.error(f"Script contains dangerous pattern: {pattern}")
                    return False
            return True
        except Exception as e:
            logger.error(f"Failed to read script: {e}")
            return False
```

### 2. 中期修复：白名单机制（优先级：P1）

实现构建命令白名单：

```python
class BuildScriptValidator:
    ALLOWED_COMMANDS = {
        'g++', 'gcc', 'clang++', 'clang',
        'ar', 'ld', 'nm', 'objcopy',
        'mkdir', 'cp', 'mv',
    }
    
    FORBIDDEN_COMMANDS = {
        'bash', 'sh', 'python', 'perl',
        'curl', 'wget', 'nc', 'telnet',
        'ssh', 'scp', 'rsync',
        'crontab', 'at', 'systemctl',
        'sudo', 'su',
    }
    
    @classmethod
    def validate(cls, script_path: str) -> bool:
        with open(script_path, 'r') as f:
            content = f.read()
        
        for line in content.split('\n'):
            line = line.strip()
            if not line or line.startswith('#'):
                continue
            
            cmd = line.split()[0] if line.split() else ''
            cmd = os.path.basename(cmd)
            
            if cmd in cls.FORBIDDEN_COMMANDS:
                raise SecurityError(f"Forbidden command: {cmd}")
        
        return True
```

## 总结

### 漏洞确认

这是一个**真实的安全风险**，与 `compile()` 漏洞属于同一漏洞模式：

1. **信任边界不清**: 用户提供的脚本被完全信任执行
2. **验证不充分**: FileChecker 只验证元数据，不验证内容
3. **prelaunch_flag 绕过**: prelaunch 模式下完全跳过输入检查
4. **可利用场景明确**: 多租户平台、共享环境、供应链攻击
5. **autotune_v2 装饰器关联**: 提供了隐蔽的攻击入口
6. **影响严重**: 任意代码执行，可能导致数据泄露、权限提升

### 修复优先级

| 优先级 | 修复方案 | 预期效果 |
|--------|----------|----------|
| P0 | 移除 prelaunch_flag 跳过检查 | 消除绕过路径 |
| P0 | 脚本内容危险模式检测 | 阻止危险命令 |
| P1 | 构建命令白名单 | 限制可执行命令 |
| P1 | 沙箱隔离执行 | 限制执行影响范围 |
| P2 | FileChecker 警告改阻止 | 减少绕过可能 |
| P3 | 文档安全警告 | 提高开发者意识 |
