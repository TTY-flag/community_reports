# 4800ec024e83：importlib加载用户控制.so文件致任意代码执行

## 漏洞概要

| 属性 | 值 |
|------|-----|
| **漏洞ID** | 4800ec024e83 |
| **漏洞类型** | 任意库加载 (CWE-114: 通过库加载注入任意代码) |
| **严重性** | High |
| **置信度** | 85% → **已确认为独立漏洞** |
| **影响版本** | MindStudio-Kernel-Launcher (当前版本) |
| **攻击复杂度** | 低 |
| **影响范围** | 以 Python 进程权限执行任意代码 |

### 漏洞摘要

MindStudio-Kernel-Launcher 的 `compile()` 函数在 `use_cache=True` 时绕过构建脚本执行，直接通过 `importlib.util.spec_from_file_location()` 加载任意 `.so` 文件。加载的库初始化代码以 Python 进程权限执行。虽然 `FileChecker` 验证文件元数据，但不验证库内容或完整性，允许攻击者加载预先放置的恶意共享库。

### 1. 库加载验证（优先级：HIGH）

在加载前实现库内容/完整性验证：

```python
import hashlib
import struct
import os

def validate_shared_library(library_path: str) -> bool:
    """
    通过 importlib 加载前验证共享库。
    执行结构和完整性检查。
    """
    
    # 1. 路径验证 - 限制为允许目录
    ALLOWED_DIRS = [
        os.path.expanduser("~/.mskl/kernels"),
        "/usr/local/lib/mskl",
        "/opt/mskl/lib",
    ]
    
    real_path = os.path.realpath(library_path)
    allowed = any(real_path.startswith(d) for d in ALLOWED_DIRS)
    if not allowed:
        raise SecurityError(f"库路径不在允许目录: {library_path}")
    
    # 2. ELF 结构验证
    with open(real_path, 'rb') as f:
        magic = f.read(4)
        if magic != b'\x7fELF':
            raise SecurityError(f"无效 ELF 魔数: {library_path}")
        
        # 检查 ELF 头异常
        elf_class = f.read(1)  # 1=32位, 2=64位
        if elf_class not in [b'\x01', b'\x02']:
            raise SecurityError(f"无效 ELF 类: {library_path}")
    
    # 3. 所有者验证（阻止，而非警告）
    file_stat = os.stat(real_path)
    if os.getuid() != 0 and os.getuid() != file_stat.st_uid:
        raise SecurityError(f"库非当前用户所有: {library_path}")
    
    # 4. 禁止符号链接（阻止，而非警告）
    if os.path.islink(library_path) or os.path.islink(real_path):
        raise SecurityError(f"不允许符号链接: {library_path}")
    
    # 5. 数字签名验证（若可用）
    # ... 若有密钥则实现签名检查
    
    return True

# 在 compile() 中应用，CompiledKernel 创建前
def compile(...):
    if use_cache and not context.autotune_in_progress:
        abs_output_bin_path = os.path.realpath(output_bin_path)
```

### 2. 允许目录白名单（优先级：HIGH）

```python
class KernelLibraryManager:
    """
    管理允许的内核库目录。
    库仅可从白名单路径加载。
    """
    
    ALLOWED_BASE_DIRS = [
        '/tmp/mskl_workspace',  # 用户工作空间（有限制）
        os.path.expanduser('~/.mskl/cache'),  # 用户缓存
    ]
```

### 漏洞确认

这是一个**真实的独立安全漏洞** (CWE-114)：

1. **独特攻击向量**：通过 `use_cache=True` 直接加载库，绕过构建脚本执行
