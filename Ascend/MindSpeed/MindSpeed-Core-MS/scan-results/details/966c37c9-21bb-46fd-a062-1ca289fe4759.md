## 漏洞描述

`tools/transfer.py` 的 `main` 函数接收多个命令行路径参数（`megatron_path`, `mindspeed_path` 等），直接用于文件操作，未进行任何安全验证。缺少以下检查：
- `os.path.realpath()` 规范化
- 路径遍历序列检查（`../`）
- 符号链接解析
- 白名单目录限制

## 修复建议

### 短期修复

**1. 添加路径验证函数**

```python
# transfer.py (新增函数)
import os
import os.path

ALLOWED_BASE_DIRS = [
    '/home/user/projects',
    '/opt/mindspeed',
    '/tmp',
]

def validate_path(path, allowed_dirs=None):
    """验证路径是否在允许的目录范围内"""
    if allowed_dirs is None:
        allowed_dirs = ALLOWED_BASE_DIRS
    
    # 1. 规范化路径（解析 .. 和符号链接）
    real_path = os.path.realpath(path)
    
    # 2. 检查路径遍历序列
    if '..' in path:
        raise ValueError(f"Path traversal detected: {path}")
    
    # 3. 检查是否在允许目录内
    for allowed_dir in allowed_dirs:
        allowed_real = os.path.realpath(allowed_dir)
        if real_path.startswith(allowed_real):
            return real_path
    
    raise ValueError(f"Path outside allowed directories: {path}")

def safe_path(path):
    """安全路径包装器"""
    try:
        return validate_path(path)
    except ValueError as e:
        logging.error(f"Invalid path: {e}")
        sys.exit(1)
```
