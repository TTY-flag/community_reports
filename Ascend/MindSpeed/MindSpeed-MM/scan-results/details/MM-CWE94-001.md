# MM-CWE94-001：TransformersModel初始化trust_remote_code启用致HuggingFace远程代码执行

**严重性**: Critical | **CWE**: CWE-94 | **置信度**: 95/100
**位置**: `mindspeed_mm/models/transformers_model.py:52-76` @ `__init__`

---

## 1. 漏洞细节

该漏洞是一个严重的代码注入漏洞，源于 HuggingFace transformers 库的 `trust_remote_code` 参数。当用户启用 `--trust-remote-code` 选项时，`AutoConfig.from_pretrained()` 和 `AutoModel.from_pretrained()` 会从 HuggingFace Hub下载并执行自定义的 Python 代码。

**关键问题**：
- HuggingFace Hub 允许模型作者提交自定义模型代码（`modeling_*.py`、`configuration_*.py`）
- 这些代码在 `trust_remote_code=True` 时会被下载并直接执行
- 攻击者可以上传包含恶意代码的模型到 HuggingFace Hub
- 用户加载该模型时，恶意代码将被自动执行

**数据流路径**：
```
CLI 参数 --trust-remote-code → args.trust_remote_code → AutoConfig.from_pretrained → HF Hub remote code execution
```

**漏洞分析**：

1. **第 52 行**：`trust_remote_code = args.trust_remote_code` - 从全局参数中获取 `trust_remote_code` 设置
2. **第 54 行**：`AutoConfig.from_pretrained(hf_path, trust_remote_code=trust_remote_code)` - 如果 `trust_remote_code=True`，将从 HuggingFace Hub 下载并执行配置代码
3. **第 69-76 行**：`model_cls.from_pretrained(..., trust_remote_code=trust_remote_code)` - 如果 `trust_remote_code=True`，将下载并执行模型代码

### HuggingFace 远程代码机制说明

当 `trust_remote_code=True` 时，transformers 库会：

1. 从 HuggingFace Hub 下载模型仓库中的自定义代码文件：
   - `modeling_<model_name>.py`
   - `configuration_<model_name>.py`
   
2. 将这些文件保存到本地缓存（通常在 `~/.cache/huggingface/`）

3. 动态导入并执行这些 Python 文件中的代码

这意味着任何上传到 HuggingFace Hub 的模型都可以包含恶意 Python 代码，并在用户加载时执行。

### 长期建议

1. **默认禁用 trust_remote_code**：
   - 将 `trust_remote_code` 默认值设为 `False`
   - 如果用户需要启用，要求明确的配置和警告提示

2. **模型来源审计**：
   - 建立可信模型来源白名单
   - 对所有外部模型进行安全审计后才能使用

3. **安全警告机制**：
   ```python
   if args.trust_remote_code:
       print_rank_0("⚠️ SECURITY WARNING: trust_remote_code is enabled!")
       print_rank_0("This will download and execute code from HuggingFace Hub.")
       print_rank_0("Only use this with models from trusted sources.")
       
       # 可选：要求用户确认
       if os.environ.get('HF_TRUST_REMOTE_CODE_CONFIRM') != 'yes':
           raise RuntimeError("Set HF_TRUST_REMOTE_CODE_CONFIRM=yes to proceed")
   ```

4. **用户教育**：
   - 在文档中明确说明 `trust_remote_code` 的安全风险
   - 提供安全模型加载的最佳实践指南
   - 建议用户只从官方或已知可信来源加载模型

5. **供应链安全**：
   - 定期扫描项目使用的模型来源
   - 对模型代码进行自动化安全检查
