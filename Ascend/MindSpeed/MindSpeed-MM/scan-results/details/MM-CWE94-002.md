# MM-CWE94-002：Tokenizer初始化trust_remote_code参数传播致远程代码执行

**严重性**: High | **CWE**: CWE-94 | **置信度**: 90/100
**位置**: `mindspeed_mm/models/text_encoder/tokenizer.py:64-73` @ `_init_tokenizer`

---

## 1. 漏洞细节

本漏洞位于 tokenizer 初始化函数 `_init_tokenizer` 中，该函数在初始化 tokenizer 时直接从 CLI 参数获取 `trust_remote_code` 设置并传递给 `AutoTokenizer.from_pretrained`。

当 `trust_remote_code=True` 时，HuggingFace transformers 库会自动下载并执行模型仓库中的自定义代码（如 `tokenization_*.py` 文件），这可能导致恶意代码执行。攻击者可以通过向模型仓库上传包含恶意 payload 的 tokenizer 代码文件，诱导用户使用 `--trust-remote-code` 参数加载该模型，从而在用户系统上执行任意代码。

### 漏洞成因

1. `trust_remote_code` 参数直接从 `get_args().trust_remote_code` 获取，未经任何安全验证
2. 该参数传递给 `AutoTokenizer.from_pretrained()`，允许执行远程自定义代码
3. 代码执行发生在 tokenizer 加载过程中，用户可能 unaware 地触发

## 2. 漏洞代码

**文件**: `mindspeed_mm/models/text_encoder/tokenizer.py` (行 53-73)

```python
def _init_tokenizer(self, module, config):
    if not isinstance(config, dict):
        config = config.to_dict()

    # Only huggingface backend is supported currently.
    self.backend = config.pop("hub_backend")
    tokenizer_name = config.pop("autotokenizer_name")
    config["pretrained_model_name_or_path"] = config.pop("from_pretrained")
    config["local_files_only"] = True
    try:
        from megatron.training import get_args
        config["trust_remote_code"] = get_args().trust_remote_code  # <-- 漏洞点：直接传递CLI参数
    except (ImportError, AssertionError, AttributeError):
        config["trust_remote_code"] = False

    if tokenizer_name in TOKENIZER_MODEL_MAPPINGS:
        tokenizer_cls = TOKENIZER_MODEL_MAPPINGS[tokenizer_name]
        return tokenizer_cls(**config)
    else:
        tokenizer_cls = getattr(module, tokenizer_name)
        return tokenizer_cls.from_pretrained(**config)  # <-- 执行远程代码
```

### 代码分析

- **行 64**: `get_args().trust_remote_code` 直接从命令行参数获取信任远程代码的设置
- **行 72-73**: 当 tokenizer_name 不在预定义映射中时，通过 `getattr(module, tokenizer_name)` 获取 transformers 类并调用 `from_pretrained(**config)`
- 此时 config 中包含 `trust_remote_code` 参数，transformers 库会据此决定是否执行远程代码

## 3. 完整攻击链路

```
[入口点] CLI --trust-remote-code 参数
↓ 用户指定 --trust-remote-code=True
[中间步骤1] megatron.training.get_args() @ training_arguments.py
↓ 返回包含 trust_remote_code 的参数对象
[中间步骤2] Tokenizer.__init__() @ tokenizer.py:40
↓ 调用 _init_tokenizer(module, config)
[中间步骤3] get_args().trust_remote_code @ tokenizer.py:64
↓ 直接传递到 config 字典
[漏洞触发] AutoTokenizer.from_pretrained(**config) @ tokenizer.py:73
↓ transformers 库下载并执行远程 tokenizer 代码
```

### 攻击链路说明

1. **入口点**: 用户通过 CLI 指定 `--trust-remote-code=True` 参数
2. **参数传递**: `get_args()` 函数返回训练参数对象，包含 `trust_remote_code` 属性
3. **配置构建**: `_init_tokenizer` 将 `trust_remote_code` 添加到 config 字典
4. **代码执行**: `AutoTokenizer.from_pretrained()` 在加载 tokenizer 时，根据 `trust_remote_code=True` 从 HuggingFace Hub 下载并执行远程 tokenizer 代码

## 4. 攻击场景

**攻击者画像**: 恶意模型仓库维护者或能够篡改 HuggingFace 模型仓库的攻击者
**攻击向量**: 通过 HuggingFace Hub 发布包含恶意 tokenizer 代码的模型
**利用难度**: 低

### 攻击步骤

1. **准备恶意模型**: 攻击者在 HuggingFace Hub 上创建一个模型仓库，包含恶意 `tokenization_*.py` 文件
2. **诱导用户加载**: 通过文档、博客、社交媒体等渠道诱导用户使用该模型进行训练或推理
3. **触发漏洞**: 用户使用 `--trust-remote-code` 参数加载模型时，tokenizer 初始化过程自动下载并执行恶意代码
4. **执行 payload**: 恶意代码可在用户系统上执行任意操作（窃取数据、建立后门、加密勒索等）
