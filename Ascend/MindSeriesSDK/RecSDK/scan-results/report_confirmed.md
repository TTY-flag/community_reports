## 5. 修复建议（基于待确认漏洞分析）

虽然本次扫描未确认任何 CONFIRMED 级别漏洞，但存在 22 个待确认漏洞需要进一步关注。以下建议基于待确认漏洞的分析，为后续安全加固提供参考：

### 优先级 1: 立即人工验证（LIKELY High 级别）

**漏洞 [VULN-SA-FILE-001] TOCTOU 配置解析漏洞**

位置: `training/tf_rec_v2/mxrec/python/config/parser.py:89-92`

问题: `file_safe_check` 在文件已打开后才调用，存在 TOCTOU（时间窗口）漏洞。

```python
# 当前代码 (lines 89-92)
with open(self._path, "r") as f:
    file_safe_check("toml path", self._path)  # 检查在 open 之后，无效
    fields = toml.load(f)
```

修复建议:
```python
# 修复方案：先检查路径，再打开文件
file_safe_check("toml path", self._path)
with open(self._path, "r") as f:
    fields = toml.load(f)
```

**漏洞 [VULN-SA-CROSS-002] 安全机制跨模块误用**

同一位置，`file_safe_check` 函数来自 `training_common` 模块，设计意图是在打开文件前验证路径，但 `tf_rec_v2` 模块调用时序错误。

建议: 在 `training_common/validator/safe_checker.py` 中增加文档说明，明确 `file_safe_check` 必须在 `open()` 之前调用。

### 优先级 2: 短期修复（路径遍历漏洞）

**漏洞 [VULN-SA-FILE-002], [VULN-DF-KVT-001], [VULN-DF-KVT-002]**

位置: `training/torch_rec_v2/dynamic_emb/dynamic_emb/distributed/key_value_table.py`

问题: 文件路径直接使用用户提供的参数，无安全检查。

```python
# 当前代码 (lines 409-412)
fkey = open(emb_key_path, "wb")
fembedding = open(embedding_file_path, "wb")
fscore = open(score_file_path, "wb")
```

修复建议:
- 在 `dump()` 和 `load()` 方法中，调用 `file_safe_check` 验证所有文件路径
- 对 JSON 文件加载使用 `class_safe_check` 验证数据结构

### 优先级 3: 计划修复（网络通信安全）

**漏洞 [VULN-SA-NET-001], [VULN-DF-LCCL-003]**

位置: `training/common/src/core/lccl/src/tools/socket/lcal_sock_exchange.cpp:284-295`

问题: Rank ID 来自网络接收，仅有范围验证，无身份认证。

```cpp
// 当前代码 (lines 283-295)
int rank = 0;
if (Recv(fd, &rank, sizeof(rank), 0) <= 0) { ... }
if (rank >= rankSize_ || rank <= 0 || clientFds_[rank] >= 0) { ... }
clientFds_[rank] = fd;
```

修复建议:
- 考虑在集群部署时实施节点身份认证（如基于预共享密钥的挑战-响应机制）
- 增加日志记录，便于事后审计异常连接行为

### 优先级 4: 配置管理加固

**漏洞 [VULN-SA-CFG-001], [VULN-DF-HCCL-001]**

位置: `training/common/python/communication/hccl/hccl_mgmt.py:98-102`

问题: `RANK_TABLE_FILE` 环境变量控制配置文件路径，无额外验证。

建议:
- 增加 `file_safe_check` 验证 rank table 文件路径
- 增加配置内容完整性校验（如签名验证）
