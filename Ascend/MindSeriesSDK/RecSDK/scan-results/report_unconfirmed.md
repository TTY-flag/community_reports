## 3. High 漏洞 (2)

### [VULN-SA-FILE-001] input_validation - TomlParser._parse_config

**严重性**: High（原评估: Medium → 验证后: High） | **CWE**: CWE-20 | **置信度**: 70/100 | **状态**: LIKELY | **来源**: security-auditor

**位置**: `training/tf_rec_v2/mxrec/python/config/parser.py:89-92` @ `TomlParser._parse_config`
**模块**: tf_rec_v2
**跨模块**: tf_rec_v2 → training_common

**描述**: File safety check is called after file is already opened, creating a TOCTOU vulnerability. In _parse_config(), the file is opened with open(self._path) before file_safe_check is called, potentially allowing symbolic link attacks.

**漏洞代码** (`training/tf_rec_v2/mxrec/python/config/parser.py:89-92`)

```c
with open(self._path, 'r') as f: file_safe_check('toml path', self._path); fields = toml.load(f)
```

**达成路径**

open(self._path) -> file_safe_check -> toml.load(f)

**验证说明**: TOCTOU vulnerability: file_safe_check is called AFTER file is opened (lines 90-91). The check 'with open(self._path, "r") as f: file_safe_check("toml path", self._path)' opens file first, then validates path. Attacker could replace file with symlink between open and check. Time window exists for symlink attack. Severity upgrade: Medium→High due to TOCTOU pattern. This is cross_module because file_safe_check from training_common is misused.

**评分明细**: base: 30 | reachability: 20 | controllability: 15 | mitigations: -10 | context: 5 | cross_file: 0
