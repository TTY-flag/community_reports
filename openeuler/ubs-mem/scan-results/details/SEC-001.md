# SEC-001: AppFreeMemory 缺少所有权校验可越权释放他人 lease

## 漏洞概述

| 属性 | 值 |
|------|-----|
| **漏洞ID** | SEC-001 |
| **类型** | Authorization Bypass (IDOR - Insecure Direct Object Reference) |
| **CWE** | CWE-639 (Authorization Bypass Through User-Controlled Key) |
| **严重性** | Critical |
| **置信度** | 85 |
| **文件** | src/mxm_lease/ipc_handler.cpp |
| **行号** | 292-392 |
| **函数** | AppFreeMemory |
| **模块** | mxm_lease |

## Codex 二次确认补充

- 结论：属实，`AppFreeMemory` 通过 name 释放 lease memory 时缺少所有权校验，属于 IDOR/越权释放。
- 场景：需要 raw IPC 能力且知道/猜到目标 lease name；普通 SDK 指针式 free 场景下不直接暴露。
- 去重：覆盖 `VULN-SEC-AUTH-001` 同类重复报告。
- 本段为二次确认补充，原报告其他内容保留不变。
## 漏洞描述

`AppFreeMemory` 函数存在典型的 **IDOR (Insecure Direct Object Reference)** 漏洞：
- 通过 `request->name_` 参数查找内存记录
- `GetUsedMemByName()` 返回包含创建者信息 (`memory.appContext.uid/gid/pid`) 的记录
- **但从未验证调用者是否是内存的所有者**
- 直接使用调用者的 `udsInfo` 进行释放操作

攻击者可释放其他用户分配的内存，导致：
1. **其他用户数据丢失**：正在使用的内存被强制释放
2. **服务中断**：其他进程的内存操作失败
3. **权限提升效果**：可执行本应只有所有者才能执行的操作

### 数据流与所有权缺失验证

```
IPC Request: AppFreeMemoryRequest
  └── request->name_ (攻击者指定 - 任意内存名称)
        └── AppFreeMemory
              ├── GetUsedMemByName(name, memory)
              │     └── 返回 memory.appContext:
              │         - memory.appContext.uid = 所有者 UID
              │         - memory.appContext.gid = 所有者 GID
              │         - memory.appContext.pid = 所有者 PID
              │
              ├── ⚠️ 应验证但未验证：
              │     if (memory.appContext.uid != udsInfo.uid ||
              │         memory.appContext.gid != udsInfo.gid) {
              │         return MXM_ERR_NOT_OWNER;
              │     }
              │
              ├── 实际使用 udsInfo (攻击者信息)：
              │     appContext.uid = udsInfo.uid;   // Line 365
              │     appContext.gid = udsInfo.gid;   // Line 366
              │     appContext.pid = udsInfo.pid;   // Line 367

              └── LeaseFree(name, isNuma)
```

### IDOR 漏洞模式

```
典型的 IDOR 漏洞模式：

┌────────────────────────────────────────────────────────────┐
│  正常流程（应有验证）：                                      │
├────────────────────────────────────────────────────────────┤
│  用户A 创建内存 → memory.appContext.uid = A               │
│  用户A 请求释放 → 验证 memory.appContext.uid == udsInfo.uid │
│                 → 验证通过 → 释放                           │
│  用户B 请求释放 → 验证 memory.appContext.uid != udsInfo.uid │
│                 → 验证失败 → 拒绝                           │
└────────────────────────────────────────────────────────────┘

┌────────────────────────────────────────────────────────────┐
│  实际流程（无验证）：                                        │
├────────────────────────────────────────────────────────────┤
│  用户A 创建内存 → memory.appContext.uid = A               │
│  用户A 请求释放 → 无验证 → 直接释放 ✓                       │
│  用户B 请求释放 → 无验证 → 直接释放 ✓ ← IDOR 漏洞！         │
│                 → 用户A 的内存被用户B 释放                  │
└────────────────────────────────────────────────────────────┘
```

## 触发条件

### 前置条件

| 条件 | 说明 | 难度 |
|------|------|------|
| **IPC Socket 访问** | 能连接到 `/var/run/ubs_mem/mem_lease` | ⭐ Easy |
| **本地进程** | Unix Domain Socket 仅支持本地连接 | ⭐ Easy |
| **内存名称已知** | 知道目标内存的名称 | ⭐ Medium |
| **无特权要求** | **任何本地用户均可调用** | ⭐ Easy |

### 内存名称获取方式

攻击者可通过以下方式获取内存名称：

1. **枚举已分配内存**：

```cpp
// AppQueryCachedMemory 无权限检查，可枚举所有缓存内存
CommonRequest request;
AppQueryCachedMemoryResponse response;
// response 包含所有内存名称列表

// 或通过其他查询接口获取
```

2. **猜测名称模式**：
   - 名称格式可能是 `lease_<size>_<timestamp>` 等
   - 通过 GenerateMemName 函数分析命名规则

3. **日志或配置泄露**：
   - 其他进程的日志可能包含内存名称
   - 配置文件中可能有硬编码的名称

### 攻击步骤

```cpp
// 攻击代码示例
#include "mxmem_lease.h"

int main() {
    auto lease = MxMemLease::GetInstance();
    
    // 1. 用户B（攻击者）获取所有内存列表
    CommonRequest queryReq;
    AppQueryCachedMemoryResponse queryResp;
    lease->QueryCachedMemory(queryReq, queryResp);
    
    // 2. 选择目标内存（属于用户A）
    for (auto& mem : queryResp.memoryList_) {
        // 假设可以通过其他方式确认这是用户A的内存
        std::string targetMemory = mem.name;  // 例如 "lease_1024_1234567890"
        
        // 3. 用户B 构造释放请求
        AppFreeMemoryRequest freeReq;
        freeReq.name_ = targetMemory;  // 用户A 的内存
        
        // 4. 用户B 发送 IPC 消息
        auto ret = lease->FreeMemory(freeReq);
        
        // ⚠️ 成功！用户A 的内存被用户B 释放
        // 用户A 的进程可能崩溃或数据丢失
        
        break;
    }
    
    return 0;
}
```

## 修复建议

### 立即修复方案

在 `AppFreeMemory` 中添加所有权验证：

```cpp
// src/mxm_lease/ipc_handler.cpp:292
int MxmServerMsgHandle::AppFreeMemory(const MsgBase* req, MsgBase* rsp, const MxmComUdsInfo& udsInfo)
{
    auto request = dynamic_cast<const AppFreeMemoryRequest*>(req);
    auto response = dynamic_cast<CommonResponse*>(rsp);
    
    if (request->name_.empty()) {
        response->errCode_ = MXM_ERR_PARAM_INVALID;
        return MXM_ERR_PARAM_INVALID;
    }

    MLSMemInfo memory;
    auto ret = MLSManager::GetInstance().GetUsedMemByName(request->name_, memory);
    if (ret != 0) {
        response->errCode_ = ret;
        return ret;
    }

    // ===== 新增: 所有权验证 =====
    
    // 1. 检查调用者是否是内存的所有者
    if (memory.appContext.uid != udsInfo.uid) {
        DBG_LOGERROR("Permission denied: memory " << request->name_ 
                     << " belongs to uid=" << memory.appContext.uid 
                     << ", caller uid=" << udsInfo.uid << " is not the owner");
        DBG_AUDITERROR("SECURITY: IDOR attack blocked in AppFreeMemory. "
                       << "memory=" << request->name_ 
                       << ", owner_uid=" << memory.appContext.uid 
                       << ", attacker_uid=" << udsInfo.uid);
        response->errCode_ = MXM_ERR_NOT_OWNER;
        return MXM_ERR_NOT_OWNER;
    }

    // 2. 可选：额外验证 gid（更严格）
    // if (memory.appContext.gid != udsInfo.gid && udsInfo.uid != 0) {
    //     return MXM_ERR_NOT_OWNER;
    // }

    // 3. root 用户可以释放任何内存（管理员权限）
    // if (udsInfo.uid == 0) {
    //     // root 特权：允许释放任何内存
    //     DBG_LOGINFO("Root user releasing memory owned by uid=" << memory.appContext.uid);
    // }

    // ===== 验证结束 =====

    // 原有逻辑继续
    // 但应使用 memory.appContext（所有者信息）而非 udsInfo
    // 因为 LeaseFree 应以所有者身份执行
    
    // ⚠️ 注意：应保持使用 memory.appContext 而非 udsInfo
    // 这样释放操作才真正属于所有者
    
    AppContext appContext;
    appContext.uid = memory.appContext.uid;  // ← 使用所有者 uid（不是 udsInfo.uid）
    appContext.gid = memory.appContext.gid;
    appContext.pid = memory.appContext.pid;
    
    auto hr = mxm::UbseMemAdapter::LeaseFree(request->name_, isNuma);
    ...
}
```

## 验证测试

### 测试用例 1: 非所有者被拒绝

```cpp
TEST(AppFreeMemory, RejectNonOwner) {
    // 用户A创建内存
    MxmComUdsInfo ownerInfo{1000, 1000, 12345};
    auto memName = CreateMemory(ownerInfo, 1024);
    
    // 用户B尝试释放
    MxmComUdsInfo attackerInfo{2000, 2000, 54321};
    AppFreeMemoryRequest request;
    request.name_ = memName;
    
    auto ret = handler.AppFreeMemory(&request, &response, attackerInfo);
    
    EXPECT_EQ(ret, MXM_ERR_NOT_OWNER);
    // 验证内存未被释放
    EXPECT_TRUE(MemoryExists(memName));
}
```

### 测试用例 2: 所有者可以释放

```cpp
TEST(AppFreeMemory, AllowOwner) {
    MxmComUdsInfo ownerInfo{1000, 1000, 12345};
    auto memName = CreateMemory(ownerInfo, 1024);
    
    AppFreeMemoryRequest request;
    request.name_ = memName;
    
    auto ret = handler.AppFreeMemory(&request, &response, ownerInfo);
    
    EXPECT_EQ(ret, MXM_OK);
    // 验证内存已被释放
    EXPECT_FALSE(MemoryExists(memName));
}
```

### 测试用例 3: root 可以释放任何内存

```cpp
TEST(AppFreeMemory, AllowRootToFreeAny) {
    MxmComUdsInfo ownerInfo{1000, 1000, 12345};
```

```cpp
TEST(AppFreeMemory, BlockIDORAttack) {
```
