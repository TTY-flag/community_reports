## Codex 二次确认补充

- 结论：属实，`AppForceFreeCachedMemory` 是全局缓存管理操作，缺少管理员/owner 授权校验。
- 场景：需要能访问 lease IPC；影响主要是 cached lease memory，cache 关闭时影响下降。
- 去重：覆盖 `VULN-SEC-AUTH-002` 同类重复报告。
- 本段为二次确认补充，原报告其他内容保留不变。
## 漏洞描述

`AppForceFreeCachedMemory` 函数允许**任意本地进程强制释放所有缓存内存**，但完全缺乏授权检查：
- 无 uid/gid 特权验证
- 无管理员权限检查
- 无调用者身份限制

该操作释放所有缓存的内存资源（通过 `DeleteAllBufferedMem()`），可能严重影响系统稳定性和其他进程的内存获取。这是一个典型的**拒绝服务 (DoS)** 漏洞。

## 攻击路径分析

### 完整调用链

```
┌─────────────────────────────────────────────────────────────────┐
│  1. 入口点: IPC Socket (Unix Domain Socket)                      │
│     路径: /var/run/ubs_mem/mem_lease                            │
│     IPC 操作码: IPC_FORCE_FREE_CACHED_MEMORY                     │
│     消息类型: CommonRequest (无特殊参数)                          │
├─────────────────────────────────────────────────────────────────┤
│  2. IPC Server Dispatch                                          │
│     ipc_server.cpp:67-70                                         │
│     IPC_FORCE_FREE_CACHED_MEMORY → IpcServer.MsgHandle          │
│     → execMap[IPC_FORCE_FREE_CACHED_MEMORY]                     │
│       = MxmServerMsgHandle::AppForceFreeCachedMemory            │
├─────────────────────────────────────────────────────────────────┤
│  3. AppForceFreeCachedMemory (ipc_handler.cpp:423-447)           │
│     ⚠️ Line 437: DeleteAllBufferedMem() - 无授权检查!             │
│     仅检查参数有效性，不验证调用者权限                              │
├─────────────────────────────────────────────────────────────────┤
│  4. MLSManager::DeleteAllBufferedMem (mls_manager.cpp:348)       │
│     释放所有缓存的内存资源                                        │
│     影响所有用户、所有进程的内存缓存                                │
└─────────────────────────────────────────────────────────────────┘
```

### 数据流

```
IPC Request (CommonRequest - 空 payload)
  └── udsInfo.uid/gid/pid (仅用于日志，不验证权限)
        └── AppForceFreeCachedMemory
              └── MLSManager::GetInstance().DeleteAllBufferedMem()
                    └── 释放所有缓存内存
                          └── 影响所有用户进程
```

### 关键代码分析

```cpp
// src/mxm_lease/ipc_handler.cpp:423-447
int MxmServerMsgHandle::AppForceFreeCachedMemory(const MsgBase* req, MsgBase* rsp, const MxmComUdsInfo& udsInfo)
{
    if (req == nullptr || rsp == nullptr) {
        return MXM_ERR_NULLPTR;  // 仅检查参数有效性
    }
    auto request = dynamic_cast<const CommonRequest*>(req);
    auto response = dynamic_cast<CommonResponse*>(rsp);
    if (request == nullptr || response == nullptr) {
        return MXM_ERR_NULLPTR;
    }

    // ⚠️ 警告：此处应该检查权限！
    // 应添加: if (udsInfo.uid != 0) { return MXM_ERR_PERMISSION_DENIED; }
    // 但代码中完全没有权限检查！

    DBG_LOGINFO("AppForceFreeCachedMemory, uid=" << udsInfo.uid << ", gid=" << udsInfo.gid << ", pid=" << udsInfo.pid);
    // 日志记录了调用者信息，但没有用于权限判断

    auto ret = MLSManager::GetInstance().DeleteAllBufferedMem();  // 直接执行！
    if (ret != 0) {
        DBG_LOGERROR("DeleteAllBufferedMem failed, res is " << ConvertErrorToString(ret));
    }
    response->errCode_ = ret;
    return ret;  // 返回结果，攻击成功
}
```

### 攻击者需要满足的条件

**几乎无门槛！**：
- 不需要 root 权限
- 不需要特定 gid（如 wheel、sudo 组）
- 不需要任何身份验证
- 只需要能连接 IPC socket

### 攻击步骤

```cpp
// 攻击代码示例 - 任何本地用户都可执行
#include "mxmem_lease.h"

int main() {
    // 1. 初始化 IPC 连接（任何用户都可以）
    auto lease = MxMemLease::GetInstance();
    
    // 2. 构造请求 - CommonRequest 是空请求
    CommonRequest request;  // 无参数！
    
    // 3. 发送 IPC 消息
    // 直接触发所有缓存内存释放
    lease->ForceFreeCachedMemory(request);
    
    // ⚠️ 成功！所有用户的缓存内存被释放
    // 其他进程的内存缓存被清空
    // 系统性能下降，可能导致服务中断
    
    return 0;
}
```

## 影响评估

### 直接影响

| 影响 | 描述 | 严重性 |
|------|------|--------|
| **DoS - 系统服务中断** | 缓存内存被清空，依赖缓存的进程性能急剧下降 | ⚠️ Critical |
| **其他用户数据丢失** | 缓存的内存数据被强制释放，可能丢失未持久化的数据 | ⚠️ High |
| **系统不稳定** | 突然释放大量内存可能触发系统异常 | ⚠️ High |
| **业务逻辑中断** | 正在复用缓存的内存分配请求失败 | ⚠️ High |

### 攻击场景分析

```
┌──────────────────────────────────────────────────────────────┐
│                    正常运行状态                                │
├──────────────────────────────────────────────────────────────┤
│  用户A: 已分配 100MB 内存，缓存中另有 50MB 可复用               │
│  用户B: 已分配 200MB 内存，缓存中另有 100MB 可复用              │
│  用户C: 正请求分配内存，准备从缓存复用                          │
│  系统: 内存缓存池正常运行                                      │
└──────────────────────────────────────────────────────────────┘
                           ↓
                   恶意用户发起攻击
                           ↓
┌──────────────────────────────────────────────────────────────┐
│                   攻击后状态                                   │
├──────────────────────────────────────────────────────────────┤
│  用户A: 已分配 100MB 内存，缓存被清空 - 无复用资源              │
│  用户B: 已分配 200MB 内存，缓存被清空 - 无复用资源              │
│  用户C: 分配请求失败 - 缓存不存在，重新分配耗时                 │
│  系统: 内存缓存池为空，需要重新构建                             │
│  ⚠️ 所有后续内存分配都需要重新创建，性能下降 10x-100x          │
└──────────────────────────────────────────────────────────────┘
```

### 影响范围

- **单机影响**：影响当前节点所有使用 ubs-mem 的进程
- **集群影响**：如果 ubs-mem 服务是多节点共享，可能影响整个集群
- **持续时间**：缓存被清空后，需要重新积累，恢复时间不确定

## 修复建议

### 立即修复方案

在 `AppForceFreeCachedMemory` 中添加 root 权限检查：

```cpp
// src/mxm_lease/ipc_handler.cpp:423
int MxmServerMsgHandle::AppForceFreeCachedMemory(const MsgBase* req, MsgBase* rsp, const MxmComUdsInfo& udsInfo)
{
    if (req == nullptr || rsp == nullptr) {
        return MXM_ERR_NULLPTR;
    }
    auto request = dynamic_cast<const CommonRequest*>(req);
    auto response = dynamic_cast<CommonResponse*>(rsp);
    if (request == nullptr || response == nullptr) {
        return MXM_ERR_NULLPTR;
    }

    // ===== 新增: 权限验证 =====
    // 1. 仅允许 root 用户 (uid == 0) 执行此操作
    if (udsInfo.uid != 0) {
        DBG_LOGERROR("Permission denied: uid=" << udsInfo.uid << " is not root");
        DBG_AUDITERROR("Unauthorized AppForceFreeCachedMemory attempt by uid=" << udsInfo.uid 
                       << ", gid=" << udsInfo.gid << ", pid=" << udsInfo.pid);
        response->errCode_ = MXM_ERR_PERMISSION_DENIED;
        return MXM_ERR_PERMISSION_DENIED;
    }

    // 2. 可选：检查特定 gid（如管理员组）
    // const gid_t ADMIN_GID = 1000; // 定义管理员组 ID
    // if (udsInfo.uid != 0 && udsInfo.gid != ADMIN_GID) {
    //     return MXM_ERR_PERMISSION_DENIED;
    // }
    // ===== 验证结束 =====

    DBG_LOGINFO("AppForceFreeCachedMemory, uid=" << udsInfo.uid << ", gid=" << udsInfo.gid << ", pid=" << udsInfo.pid);
    auto ret = MLSManager::GetInstance().DeleteAllBufferedMem();
    ...
}
```

### 长期改进方案

1. **角色权限系统 (RBAC)**：
   - 定义管理员角色
   - 细粒度权限控制（读、写、管理）
   - 配置文件驱动的权限策略

```cpp
// 权限配置示例 (permission_config.yaml)
permissions:
  force_free_cache:
    allowed_roles: [admin, operator]
    allowed_uids: [0]
    allowed_gids: [wheel, sudo, mxm-admin]
```

2. **安全策略模块**：
   - 独立的权限验证模块
   - 可动态加载的策略引擎
   - 支持 SELinux/AppArmor 集成

3. **速率限制**：
   - 即使是管理员，也限制调用频率
   - 防止意外或恶意重复调用

```cpp
// 速率限制示例
static std::atomic<int64_t> lastForceFreeTime{0};
const int64_t MIN_INTERVAL_MS = 60000; // 最少间隔 60 秒

auto now = GetCurrentTimeMs();
if (now - lastForceFreeTime.load() < MIN_INTERVAL_MS) {
    DBG_LOGERROR("Force-free-cache rate limited. Last call was " 
                 << (now - lastForceFreeTime.load()) << "ms ago.");
    return MXM_ERR_RATE_LIMITED;
}
lastForceFreeTime.store(now);
```
