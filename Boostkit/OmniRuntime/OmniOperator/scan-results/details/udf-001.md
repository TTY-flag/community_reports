# udf-001：UDF信任边界违规漏洞

## 漏洞概述

此漏洞与 **VULN-XMOD-001** 和 **VULN-SEC-UDF-001** 属于同一攻击链。

详细分析请参考：
- `VULN-XMOD-001.md` - 完整攻击链分析
- `VULN-SEC-UDF-001.md` - 关键代码分析

## 漏洞核心

从 C++ 代码生成层到 Java UDF 执行层的数据流存在信任边界违规：

```
SQL 查询 (用户控制)
    ↓
【信任边界 1】JSONParser 解析用户表达式
    ↓
funcName 存储在 FuncExpr 中
    ↓
【信任边界 2】expression_codegen.cpp 编译表达式
    ↓
EvaluateHiveUdfSingle 接收 funcName 作为 udfClass
    ↓
【信任边界 3】ExecuteHiveUdfSingle 传递到 JNI
    ↓
【信任边界 4】Java Class.forName 加载类
    ⚠️ 无验证跨越所有信任边界
```
