# JNI-005: JNI 空指针解引用漏洞

## 漏洞概述

**漏洞类型**: 空指针解引用 (CWE-476)  
**严重级别**: High  
**置信度**: 85%  
**影响模块**: jni_bindings, operator

OmniOperator 的 JNI 函数 `getOutputNative` 在获取 `nativeOperator` 指针后立即调用 `GetOutput`，未进行空指针检查。与 JNI-001 漏洞模式相同。

## 关键代码分析

### jni_operator.cpp:178-198

```cpp
JNIEXPORT jobject JNICALL Java_nova_hetu_omniruntime_operator_OmniOperator_getOutputNative(
    JNIEnv *env, jobject jObj, jlong jOperatorAddr)
{
    std::call_once(loadVecBatchClsFlag, LoadVecBatchAndOmniResults, env);
    ...
    
    // ⚠️ 无空指针检查，直接转换
    auto *nativeOperator = reinterpret_cast<op::Operator *>(jOperatorAddr);
    VectorBatch *outputVecBatch = nullptr;
    
    JNI_METHOD_START
    // ⚠️ 立即调用方法，无有效性验证
    nativeOperator->GetOutput(&outputVecBatch);
    JNI_METHOD_END_WITH_VECBATCH(nullptr, outputVecBatch)
    ...
}
```

## 攻击模式

与 JNI-001 相同，恶意 Java 代码可传入无效地址：
- `jOperatorAddr = 0` → 空指针解引用
- `jOperatorAddr = 0xdeadbeef` → 无效地址访问

## 修复建议

参考 JNI-001 的修复方案，添加空指针检查和有效性验证。
