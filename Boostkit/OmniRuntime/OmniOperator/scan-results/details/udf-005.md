# udf-005: 不安全类加载漏洞

## 漏洞概述

**漏洞类型**: 不安全类加载 (CWE-94)  
**严重级别**: High  
**置信度**: 85%  
**影响模块**: udf (Java)

OmniOperator 的 `HiveUdfExecutor.java` 在加载 UDF 类时使用 `Class.forName(udfClassName, true, loader)`，其中 `initialize=true` 参数导致类的静态初始化器在类型检查之前执行。攻击者可以通过恶意类的静态初始化器执行任意代码，即使类不实现合法的 UDF 接口。

## 漏洞触发条件

1. 攻击者能够控制 `udfClassName` 参数
2. 攻击者能够在类加载路径放置恶意类文件
3. 恶意类包含静态初始化器（static initializer）

## 关键代码分析

### HiveUdfExecutor.java:163-194

```java
private static void executeSingle(URLClassLoader loader, String udfClassName, ...) {
    Method method;
    Object udfObj;
    try {
        // ⚠️ 关键缺陷: initialize=true 导致静态初始化器立即执行
        Class<?> udfClass = Class.forName(udfClassName, true, loader);
        
        // ⚠️ UDF 接口检查在类加载之后执行，此时静态初始化器已完成
        if (UDF.class.isAssignableFrom(udfClass)) {
            method = findValidMethod(udfClass, paramTypes, retType);
            udfObj = udfClass.getConstructor().newInstance();
        } else {
            String msg = "Not support " + udfClassName + " now.";
            throw new OmniRuntimeException(..., msg);
        }
    } catch (...) {
        // 即使检查失败，静态初始化器代码已执行
        ...
    }
    ...
}
```

### 同样的问题存在于 executeBatch (line 370)

```java
Class<?> udfClass = Class.forName(udfClassName, true, loader);  // ⚠️ 同样的问题
if (UDF.class.isAssignableFrom(udfClass)) {  // ⚠️ 检查太晚
    ...
}
```

**安全缺陷分析**:
- `Class.forName(className, true, loader)` 中 `true` 表示初始化类
- 类初始化会执行静态初始化块 `{ ... }`
- `UDF.class.isAssignableFrom(udfClass)` 检查在初始化之后
- 即使类不继承 `UDF`，静态初始化器已执行完毕

## 危害评估

### 攻击影响
- **任意代码执行**: 静态初始化器可执行任意 Java 代码
- **系统命令执行**: `Runtime.exec()` 可执行系统命令
- **文件系统访问**: 可读取/写入任意文件
- **网络连接**: 可建立外部连接（反向 Shell）

### CVSS 评分预估
- **攻击向量**: Network (通过 SQL/配置)
- **攻击复杂度**: Low (构造简单恶意类)
- **权限要求**: Low (SQL 执行权限)
- **影响**: High (完全代码执行)

**预估 CVSS 评分**: 9.0 (Critical)

## 修复建议

### 1. 使用 initialize=false 先检查后加载（优先级：高）

```java
private static void executeSingle(URLClassLoader loader, String udfClassName, ...) {
    Method method;
    Object udfObj;
    try {
        // 关键修复: 先加载但不初始化
        Class<?> udfClass = Class.forName(udfClassName, false, loader);
        
        // 在初始化前检查接口
        if (!UDF.class.isAssignableFrom(udfClass)) {
            throw new OmniRuntimeException(..., 
                "Class does not extend UDF: " + udfClassName);
        }
        
        // 添加白名单检查
        if (!isAllowedUdfClass(udfClassName)) {
            throw new OmniRuntimeException(...,
                "Unauthorized UDF class: " + udfClassName);
        }
        
        // 安全检查通过后才初始化
        method = findValidMethod(udfClass, paramTypes, retType);
        udfObj = udfClass.getConstructor().newInstance();  // 安全初始化
        
    } catch (...) {
        ...
    }
    ...
}
```

### 2. 添加 UDF 类白名单（优先级：高）

```java
private static final Set<String> ALLOWED_UDF_CLASSES = Set.of(
    "org.apache.hadoop.hive.ql.udf.UDFType...",
    // 预定义的合法 UDF 类
);

private static boolean isAllowedUdfClass(String className) {
    return ALLOWED_UDF_CLASSES.contains(className);
}
```

### 3. 类名格式验证（优先级：中）

```java
private static boolean isValidClassName(String className) {
    // 验证类名格式
    if (className == null || className.isEmpty()) {
        return false;
    }
    // 不允许危险字符
    if (className.contains("..") || className.contains("/") || 
        className.startsWith("java.") || className.startsWith("javax.")) {
        return false;
    }
    // 检查标准 Java 类名格式
    return className.matches("[a-zA-Z_$][a-zA-Z0-9_$]*(\\.[a-zA-Z_$][a-zA-Z0-9_$]*)*");
}
```

### 4. 使用 SecurityManager（优先级：中）

```java
// 在 UDF 执行前设置安全策略
SecurityManager sm = new UdfSecurityManager();
System.setSecurityManager(sm);
try {
    // 执行 UDF
} finally {
    System.setSecurityManager(null);
}
```
