## Vulnerability Description

The `setenv()` function is called with the `local_dirs` parameter received from the JVM without any validation or sanitization. This allows an attacker who can control the Spark configuration to manipulate the `NATIVESQL_SPARK_LOCAL_DIRS` environment variable, potentially redirecting file operations to attacker-controlled paths.

### Vulnerable Code

```cpp
// File: cpp-omni/src/jni/SparkJniWrapper.cpp
// Lines: 103-105

auto local_dirs = env->GetStringUTFChars(local_dirs_jstr, JNI_FALSE);
setenv("NATIVESQL_SPARK_LOCAL_DIRS", local_dirs, 1);
env->ReleaseStringUTFChars(local_dirs_jstr, local_dirs);
```

The code directly uses the `local_dirs` string from the JVM without:
1. Path validation (checking for `..` sequences)
2. Path canonicalization
3. Whitelist validation of allowed directories
4. Checking for absolute vs relative paths

## Attack Vectors

### 1. Path Traversal Attack

An attacker who can control the `spark.local.dir` configuration could inject paths containing:
- `../` sequences to escape intended directories
- Absolute paths to sensitive system locations

**Example malicious input:**
```
/tmp/legitimate,/etc/cron.d,/root/.ssh
```

Or path traversal:
```
/var/spark/./../../../etc
```

### 2. Arbitrary File Write

If an attacker controls the configuration (e.g., through compromised Spark job submission, malicious configuration injection, or cluster management tool exploitation), they could:
1. Write shuffle spill files to arbitrary locations
2. Overwrite sensitive system files
3. Create files in locations that could lead to privilege escalation

### 3. Symlink Attacks

The code uses `mkdir()` without `O_NOFOLLOW`, potentially allowing symlink-based attacks:
```cpp
mkdir(dir.c_str(), S_IRWXU|S_IRWXG|S_IROTH|S_IXOTH);
```

## Root Cause Analysis

1. **Missing Input Validation**: The `local_dirs` string is passed through the JNI boundary without any validation or sanitization.

2. **Process-wide Environment Variable**: Using `setenv()` affects the entire process, not just the current thread/shuffle operation. This is a global side effect.

3. **No Path Canonicalization**: Paths are not resolved to their canonical forms before use, allowing `..` sequences and symbolic links to escape intended directories.

4. **No Whitelist Enforcement**: There's no check to ensure directories are within expected bounds (e.g., under `/tmp` or configured Spark directories).

## Remediation Recommendations

### 1. Input Validation (Recommended)

```cpp
// In SparkJniWrapper.cpp
auto local_dirs = env->GetStringUTFChars(local_dirs_jstr, JNI_FALSE);
std::string dirs(local_dirs);

// Validate paths
std::vector<std::string> validated_dirs;
std::stringstream ss(dirs);
std::string dir;
while (std::getline(ss, dir, ',')) {
    // Remove leading/trailing whitespace
    dir.erase(0, dir.find_first_not_of(" \t\n\r"));
    dir.erase(dir.find_last_not_of(" \t\n\r") + 1);
    
    // Reject paths containing traversal sequences
    if (dir.find("..") != std::string::npos) {
        env->ThrowNew(runtimeExceptionClass, "Invalid local_dirs: path traversal detected");
        env->ReleaseStringUTFChars(local_dirs_jstr, local_dirs);
        return 0;
    }
    
    // Reject relative paths (require absolute paths)
    if (dir.empty() || dir[0] != '/') {
        env->ThrowNew(runtimeExceptionClass, "Invalid local_dirs: relative paths not allowed");
        env->ReleaseStringUTFChars(local_dirs_jstr, local_dirs);
        return 0;
    }
    
    // Canonicalize path
    char resolved[PATH_MAX];
    if (realpath(dir.c_str(), resolved) != nullptr) {
        validated_dirs.push_back(resolved);
    }
}
```

### 2. Pass Validated Paths Directly

Instead of using environment variables, pass validated paths directly:

```cpp
// Instead of:
setenv("NATIVESQL_SPARK_LOCAL_DIRS", local_dirs, 1);

// Pass to Splitter via options:
splitOptions.local_dirs = validated_dirs;
```

### 3. Add Whitelist Check

```cpp
// Define allowed base directories
const std::vector<std::string> ALLOWED_BASES = {
    "/tmp",
    "/var/tmp",
    "/var/lib/spark"
};

bool isPathAllowed(const std::string& path) {
    char resolved[PATH_MAX];
    if (realpath(path.c_str(), resolved) == nullptr) return false;
    
    for (const auto& base : ALLOWED_BASES) {
        if (strncmp(resolved, base.c_str(), base.length()) == 0) {
            return true;
        }
    }
    return false;
}
```

### 4. Avoid Global State

Move away from using environment variables to pass configuration:
- Use class member variables
- Pass configuration through function parameters
- Use thread-local storage if needed
